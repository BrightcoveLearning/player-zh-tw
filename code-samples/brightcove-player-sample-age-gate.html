開發順序

這是推薦的開發順序：

  1. 使用頁內嵌入播放器實現來測試播放器，插件和CSS的功能（如果需要CSS）
  2. 將插件的JavaScript和CSS放入單獨的文件中以進行本地測試
  3. 解決任何錯誤後，將插件代碼和CSS部署到服務器上
  4. 使用Studio將插件和CSS添加到播放器
  5. 如果您確定iframe實現更合適，請替換頁內嵌入播放器實現（在下一部分中詳細介紹）

有關這些步驟的詳細信息，請查看一步步：插件開發指導。

iframe或頁內嵌入

在開發Brightcove Player的增強功能時，您需要確定代碼是否最適合iframe或頁內嵌入實施。最佳實踐建議是構建一個可與iframe實現配合使用的插件。使用iframe播放器的優點是：

  • 與現有的JavaScript和/或CSS無衝突
  • 自動響應
  • iframe可簡化在社交媒體應用中的使用（或需要將視頻“傳播"到其他應用中的任何時候）

儘管集成頁內嵌入播放器可能會更加複雜，但是有時您會圍繞該實現計劃代碼。概括地說，當包含頁面需要與播放器進行通信時，這種方法是最好的。具體來說，以下是一些示例：

  • 包含頁面中的代碼需要偵聽播放器事件並對其進行操作
  • 播放器使用包含頁面中的樣式
  • iframe將導致應用邏輯失敗，例如從包含頁面的重定向

即使最終實現未使用iframe代碼，您仍可以將頁內嵌入代碼與JavaScript插件和CSS單獨文件一起使用。這封裝了你的邏輯，以便您可以輕鬆地在多個玩家中使用它。

應用程式樣式

CSS將覆蓋物和年齡門文本放置在覆蓋物中。還有一種隱藏控制欄和播放按鈕的樣式。

隱藏播放器控件

讓我們仔細看看如何使用CSS隱藏大播放按鈕和控制欄。有關更改Brightcove Player外觀的介紹，請參見自定義玩家外觀指導。

首先，讓我們更改樣式以隱藏播放器的大播放按鈕。

  1. 在Chrome瀏覽器中，打開開發者工具。
  2. 右鍵單擊播放器，然後選擇檢查元素。
  3. 在裡面元素的部分開發者工具，選擇與大播放按鈕關聯的HTML。
  4. 在裡面款式部分，您可以看到原始的display值為的屬性block設置在<div>元素。
  5. 覆蓋CSS值時，特定樣式會勝過特定樣式。由於元素的最低特異性值，我們可以通過簡單地指定vjs-big-play-button班級。

  6. 在下面的大播放按鈕中查看我們的CSS。在此示例中，我們說選擇所有類名稱為的元素vjs-big-play-button類別為的元素的後繼者video-agelist。請記住，在JavaScript代碼中，我們添加了video-agelist給玩家上課。

                  video-agelist.video-js .vjs-control-bar,
              .video-agelist .vjs-big-play-button {
                  display: none;
              }

接下來，讓我們更改樣式以隱藏播放器的控制欄。

  1. 在裡面元素的部分開發者工具，選擇與播放器控制欄關聯的HTML。
  2. 在裡面款式部分，您可以看到原始的display值為的屬性flex在選擇器中設置了三個類屬性。
  3. 要覆蓋控制欄的樣式，我們在選擇器中至少需要三個類屬性，包括vjs-control-bar班級。
  4. 在下面的控制欄上查看我們的CSS。在此示例中，我們說選擇所有類名稱為的元素vjs-control-bar這是元素的後裔，兩個元素的類名均為video-agelist和一個類名video-js。

插件代碼

通常，將JavaScript轉換為Brightcove Player插件時，需要進行名義上的更改。一項必需的更改是替換標準的使用ready()方法與定義插件的代碼。

這是可以與播放器一起使用的非常常用的JavaScript代碼開始：

videojs.getPlayer('myPlayerID').ready(function() {
  var myPlayer = this;
  ...
});

您將更改第一行以使用標準語法來啟動Brightcove Player插件：

videojs.registerPlugin('pluginName', function(options) {
  var myPlayer = this;
  ...
});

如前所述，您可以在該文檔的相應GitHub存儲庫中看到該插件的JavaScript代碼： age-gate.js。
