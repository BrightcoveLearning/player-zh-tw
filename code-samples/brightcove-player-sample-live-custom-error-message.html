提供此示例和相關代碼作為生產開發的指南。該代碼未經認證可用於生產用途，也未經官方支持。

播放器將無法連接到實時流。創建一個ModalDialog對象，在其中放置一個自定義圖像，然後顯示ModalDialog。如果關閉了ModalDialog，則顯示默認錯誤。

開發順序

這是推薦的開發順序：

  1. 使用頁內嵌入播放器實現來測試播放器，插件和CSS的功能（如果需要CSS）
  2. 將插件的JavaScript和CSS放入單獨的文件中以進行本地測試
  3. 解決任何錯誤後，將插件代碼和CSS部署到服務器上
  4. 使用Studio將插件和CSS添加到播放器
  5. 如果您確定iframe實現更合適，請替換頁內嵌入播放器實現（在下一部分中詳細介紹）

有關這些步驟的詳細信息，請查看一步步：插件開發指導。

iframe或頁內嵌入

在開發Brightcove Player的增強功能時，您需要確定代碼是否最適合iframe或頁內嵌入實施。最佳實踐建議是構建一個可與iframe實現配合使用的插件。使用iframe播放器的優點是：

  • 與現有的JavaScript和/或CSS無衝突
  • 自動響應
  • iframe可簡化在社交媒體應用中的使用（或需要將視頻“傳播"到其他應用中的任何時候）

儘管集成頁內嵌入播放器可能會更加複雜，但是有時您會圍繞該實現計劃代碼。概括地說，當包含頁面需要與播放器進行通信時，這種方法是最好的。具體來說，以下是一些示例：

  • 包含頁面中的代碼需要偵聽播放器事件並對其進行操作
  • 播放器使用包含頁面中的樣式
  • iframe將導致應用邏輯失敗，例如從包含頁面的重定向

即使最終實現未使用iframe代碼，您仍可以將頁內嵌入代碼與JavaScript插件和CSS單獨文件一起使用。這封裝了你的邏輯，以便您可以輕鬆地在多個玩家中使用它。

申請流程

此應用程序背後的基本邏輯是：

  • 創建ModalDialog的實例，然後在其中放置一個自定義圖像。
  • 偵聽錯誤，如果錯誤類型正確，則顯示自定義ModalDialog。
  • 如果視頻的觀看者選擇關閉自定義ModalDialog，則顯示默認錯誤消息。

創建一個HTML div元素

找到標有以下代碼的代碼：

// ### Create a div in which to place HTML content and place image inside ###

您創建一個HTML div元素，然後在實時供稿無效的情況下在其中顯示您希望顯示的圖像。

建立一個對象，其中的HTML div元素將被放置

找到標有以下代碼的代碼：

  // ### Define options object to be used for modal content ###

當您創建ModalDialog時，這是必需的。

創建ModalDialog並將圖像作為其內容

找到標有以下代碼的代碼：

// ### Create Modal with options object ###

這是創建ModalDialog實例並將其添加到播放器的標準代碼。默認情況下不顯示。

偵聽要發送的錯誤

找到標有以下代碼的代碼：

// ### Listen for an error event ###

調度錯誤後，獲取錯誤代碼和持續時間。

檢查錯誤類型

找到標有以下代碼的代碼：

  // ### Check if the error code and duration mean no video has loaded ###

要檢查直播是否未播放，請使用以下命令：

if (((errNo == "4" && duration == 0) || (errNo == "4" && Number.isNaN(duration)))) { // Hide the error display myPlayer.errorDisplay.hide(); // If conditions met show the custom modal myModal.open(); }

條件檢查errNo == "4" && 持續時間 == 0適用於 Chrome 和 Firefox。第二個條件檢查errNo == "4" && Number.isNaN(duration)需要 Safari 和 Android 上的瀏覽器。

顯示默認錯誤是自定義ModalDialog已關閉

找到標有以下代碼的代碼：

// ### If custom modal error closed, show the default error ###

僅當您希望實現此行為時，才執行此操作。只需聽modalclose事件及其事件處理程序中顯示默認錯誤。

插件代碼

通常，將JavaScript轉換為Brightcove Player插件時，需要進行名義上的更改。一項必需的更改是替換標準的使用ready()方法與定義插件的代碼。

這是可以與播放器一起使用的非常常用的JavaScript代碼開始：

videojs.getPlayer('myPlayerID').ready(function() {
  var myPlayer = this;
  ...
  });

您將更改第一行以使用標準語法來啟動Brightcove Player插件：

videojs.registerPlugin('pluginName', function(options) {
  var myPlayer = this;
  ...
  });

如前所述，您可以在該文檔的相應GitHub存儲庫中看到該插件的JavaScript代碼： liveCustomError.js。
