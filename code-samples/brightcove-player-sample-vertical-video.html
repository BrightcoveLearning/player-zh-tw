開發順序

這是推薦的開發順序：

  1. 使用頁內嵌入播放器實現來測試播放器，插件和CSS的功能（如果需要CSS）
  2. 將插件的JavaScript和CSS放入單獨的文件中以進行本地測試
  3. 解決任何錯誤後，將插件代碼和CSS部署到服務器上
  4. 使用Studio將插件和CSS添加到播放器
  5. 如果您確定iframe實現更合適，請替換頁內嵌入播放器實現（在下一部分中詳細介紹）

有關這些步驟的詳細信息，請查看一步步：插件開發指導。

iframe或頁內嵌入

在開發Brightcove Player的增強功能時，您需要確定代碼是否最適合iframe或頁內嵌入實施。最佳實踐建議是構建一個可與iframe實現配合使用的插件。使用iframe播放器的優點是：

  • 與現有的JavaScript和/或CSS無衝突
  • 自動響應
  • iframe可簡化在社交媒體應用中的使用（或需要將視頻“傳播"到其他應用中的任何時候）

儘管集成頁內嵌入播放器可能會更加複雜，但是有時您會圍繞該實現計劃代碼。概括地說，當包含頁面需要與播放器進行通信時，這種方法是最好的。具體來說，以下是一些示例：

  • 包含頁面中的代碼需要偵聽播放器事件並對其進行操作
  • 播放器使用包含頁面中的樣式
  • iframe將導致應用邏輯失敗，例如從包含頁面的重定向

即使最終實現未使用iframe代碼，您仍可以將頁內嵌入代碼與JavaScript插件和CSS單獨文件一起使用。這封裝了你的邏輯，以便您可以輕鬆地在多個玩家中使用它。

播放器/ HTML配置

本節詳細介紹了播放器創建過程中所需的任何特殊配置。此外，還介紹了除頁面內嵌入播放器實現代碼外，還必須添加到頁面的其他HTML元素。

播放程式組態

您為此示例創建的Brightcove Player不需要任何特殊配置。

視頻配置

本示例使用在移動設備上以縱向模式創建的視頻資產。已使用標準的攝取配置文件攝取了此垂直視頻。

HTML內聯樣式

這<video>標籤的style屬性設置播放器的大小以填充其父級<div>元素。

他們倆<div>周圍的元素<video>標籤用於創建最大寬度為286px。

申請流程

如果您對使用HTML樣式的播放器大小感到滿意，則可以在沒有以下CSS和JavaScript代碼的情況下實現播放器。

可選的：如果您希望播放器在播放開始時擴展到設備視圖的整個寬度，然後返回到原始位置，則需要附加代碼286px視頻結束時的寬度。基本邏輯是：

  • 初始化變量並檢查移動設備
  • 定義模式關閉按鈕
  • 播放開始時，進入全屏模式
  • 播放結束後，退出全屏模式

初始化變量並檢查移動設備

找到標有以下代碼的代碼：

// +++ Initialize variables and check for mobile device +++

檢查是否在移動設備上正在播放。

定義模式關閉按鈕

找到標有以下代碼的代碼：

// +++ Define a modal close button +++

如果用戶在移動設備上，則定義一個關閉按鈕以退出全角模式。播放器將在視頻播放結束時或在用戶單擊關閉模式按鈕時退出全寬模式。

播放開始時，進入全屏模式

找到標有以下代碼的代碼：

// +++ When playback begins, enter full width mode +++

視頻播放開始時，將播放器設置為設備視圖的整個寬度。

播放結束後，退出全屏模式

找到標有以下代碼的代碼：

// +++ When playback ends, exit full width mode +++

視頻播放結束時，將播放器設置為原始寬度。

應用程式樣式

CSS為關閉按鈕和大播放按鈕設置樣式。

插件代碼

通常，將JavaScript轉換為Brightcove Player插件時，需要進行名義上的更改。一項必需的更改是替換標準的使用ready()方法與定義插件的代碼。

這是可以與播放器一起使用的非常常用的JavaScript代碼開始：

videojs.getPlayer('myPlayerID').ready(function() {
  var myPlayer = this;
  ...
});

您將更改第一行以使用標準語法來啟動Brightcove Player插件：

videojs.registerPlugin('pluginName', function(options) {
  var myPlayer = this;
  ...
});

如前所述，您可以在該文檔的相應GitHub存儲庫中看到該插件的JavaScript代碼： vertical-video.js。
