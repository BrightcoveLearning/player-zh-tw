播放器示例

提供此示例和相關代碼作為生產開發的指南。該代碼未經認證可用於生產用途，也未經官方支持。

在此示例中，您將在設置的時間後看到覆蓋消息。該示例將在6秒內顯示覆蓋圖，但是時間是可配置的。如果查看者在指定時間內單擊該消息，則繼續播放。如果不是，則視頻暫停。如果您有較長的內容，則此技術很有用。這樣一來，您就不會浪費帶寬開始觀看視頻的觀眾，而不必花心思就停止播放視頻，然後再去做其他事情。

此示例使用位於以下位置的開源Video.js插件https://github.com/TylerZubke/Videojs-AFK-Monitor。本文檔中的示例代碼將利用此插件。

開發順序

這是推薦的開發順序：

  1. 使用頁內嵌入播放器實現來測試播放器，插件和CSS的功能（如果需要CSS）
  2. 將插件的JavaScript和CSS放入單獨的文件中以進行本地測試
  3. 解決任何錯誤後，將插件代碼和CSS部署到服務器上
  4. 使用Studio將插件和CSS添加到播放器
  5. 如果您確定iframe實現更合適，請替換頁內嵌入播放器實現（在下一部分中詳細介紹）

有關這些步驟的詳細信息，請查看一步步：插件開發指導。

iframe或頁內嵌入

在開發Brightcove Player的增強功能時，您需要確定代碼是否最適合iframe或頁內嵌入實施。最佳實踐建議是構建一個可與iframe實現配合使用的插件。使用iframe播放器的優點是：

  • 與現有的JavaScript和/或CSS無衝突
  • 自動響應
  • iframe可簡化在社交媒體應用中的使用（或需要將視頻“傳播"到其他應用中的任何時候）

儘管集成頁內嵌入播放器可能會更加複雜，但是有時您會圍繞該實現計劃代碼。概括地說，當包含頁面需要與播放器進行通信時，這種方法是最好的。具體來說，以下是一些示例：

  • 包含頁面中的代碼需要偵聽播放器事件並對其進行操作
  • 播放器使用包含頁面中的樣式
  • iframe將導致應用邏輯失敗，例如從包含頁面的重定向

即使最終實現未使用iframe代碼，您仍可以將頁內嵌入代碼與JavaScript插件和CSS單獨文件一起使用。這封裝了你的邏輯，以便您可以輕鬆地在多個玩家中使用它。

播放器/ HTML配置

本節詳細介紹了播放器創建過程中所需的任何特殊配置。此外，還介紹了除頁面內嵌入播放器實現代碼外，還必須添加到頁面的其他HTML元素。

播放程式組態

您為此示例創建的Brightcove Player不需要任何特殊配置。

其他HTML

包含第三方插件的CSS和JavaScript文件。

申請流程

此應用程序背後的基本邏輯是：

  • 這useractive和userinactive監視事件以跟踪時間。
  • 在指定的時間之後顯示覆蓋圖，以查看客戶是否仍在觀看視頻。
  • 在指定的時間內顯示疊加層且未單擊文本時，請暫停視頻。
  • 單擊該按鈕時，或者觀眾在暫停後開始播放視頻時，請重新初始化值並重新開始時間跟踪。

本示例使用插件

使用最佳做法調用該插件，如下所示

  <script>
    videojs.getPlayer('myPlayerID').ready(function() {
      var myPlayer = this;
      var options = {'showMessageAfter': 1000, 'pausePlayerAfter': 1000};
      myPlayer.AFKMonitor(options);
    });
  </script>

配置插件

可以使用許多選項來自定義插件。一些細節是：

  • 有關配置選項的完整列表，請參見插件的自述文件文件。
  • 這showMessageAfter（以毫秒為單位）確定按鈕顯示之前的時間。
    請注意，由於插件使用userinactive事件，時間由showMessageAfter將有 5 秒添加到它。這是因為userinactive事件在用戶不活動 5 秒後調度。
  • 這pausePlayerAfter（以毫秒為單位）選項也很有幫助。這是您允許用戶在播放器暫停之前單擊按鈕的時候。這使用戶可以在不中斷視頻播放的情況下單擊按鈕。
如果您希望將此插件與 Brightcove Live 一起使用，則應使用位於以下位置的修改後的插件https://github.com/BrightcoveLearning/18189-are-you-still-watching/blob/master/plugin/videojs.afk-monitor-live.js .

插件代碼

您將需要克隆存儲庫以訪問代碼，然後將其放置在Internet上可訪問的位置以供使用。

您需要更新插件的 JavaScript 代碼。在倉庫中，以下已過時的代碼用於註冊插件： 
  videojs.plugin('AFKMonitor', AFKMonitor);

您應該用以下代碼替換該行代碼：

  videojs.registerPlugin('AFKMonitor', AFKMonitor);

如前所述，您可以在該文檔的相應GitHub存儲庫中看到該插件的JavaScript代碼： videojs.afk-monitor.js。

與播放器一起使用插件

將插件的CSS和JavaScript文件存儲在Internet上可訪問的位置後，即可將其與播放器一起使用。在工作室的球員模塊你可以選擇一個播放器，然後在插件部分將 URL 添加到 CSS 和 JavaScript 文件，並添加姓名和選項 , 如果需要選項。

有關使用 Studio 為玩家分配插件的更多信息，請參閱配置播放器插件文檔。
