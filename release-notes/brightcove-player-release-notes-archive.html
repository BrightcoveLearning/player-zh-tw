筆記：默認情況下，更新會自動應用到所有播放器。為防止播放器自動更新，您必須將其設置為手動更新（也稱為鎖定播放器）。您可以在Studio Players模塊（外部站點）或使用播放器管理API。

選擇不使用自動更新應該是一種臨時措施，可以在解決更新造成的任何問題時應用。如果您不將播放器更新為當前版本，則會錯過新功能和錯誤修復，並且某些時候將不再支持播放器的舊版本。

還，您可以在Brightcove Player發行公告 Google網上論壇。

1 It should be noted that, in this context, making the player "responsive to browser size by default" is distinct from making the player fluid.以下內容應闡明新功能：

  • 需要明確的是：靜態玩家不會突然變得流暢！
  • 在這種情況下使用“響應"意味著默認情況下我們已在Brightcove Player中啟用了新的Video.js響應選項（而Video.js中則默認為關閉）。此選項將導致vjs-layout-*每當更改播放器的大小時，將基於默認的斷點集將其應用於播放器類。這意味著“響應性"實際上不是基於瀏覽器視口的大小，而是基於嵌入式播放器的大小。
  • 基於此新Video.js選項進行了兩個小的UI調整。可以預期，這些調整應該不會影響許多用戶（如果有的話），但是出於完整性考慮，它們是：
    • 字幕設置對話框
    • 社交插件對話框
  • 從 Studio 或通過 Video.js 進行響應式嵌入的用戶流體選項將能夠利用新的vjs-layout-*類以根據播放器大小改進其播放器 UI 自定義。嵌入尺寸為靜態的用戶仍將獲得vjs-layout-*類適用於他們的播放器，但除非播放器的大小在運行時更改，否則它不會更改。
  • 對於小寬度播放器實現（例如425像素），控制欄中的某些按鈕可能會被隱藏。儘管您將鬆開響應式功能，但以下代碼將防止按鈕被隱藏：

新的功能

Brightcove Player版本5在不使用Flash的情況下對HLS視頻提供了新的支持。在此版本中，在所有支持Media Source Extensions API（MSE）的瀏覽器中，Brightcove Player都可以播放HLS視頻沒有閃光燈！這意味著在Firefox，Chrome，Safari或Microsoft Edge上不使用Flash。Flash仍將用於IE的較早版本，但可以通過配置禁用。

MSE HLS具有幾個巨大的好處：

  1. 即使在高分辨率和高幀率下，視頻也可以快速加載並流暢播放，包括4K和60FPS視頻-在適度的硬件上
  2. 它大約使用-50％的CPU負載來渲染相同的視頻，這意味著電池可以使用更長的時間，而筆記本電腦的風扇將無法打開
  3. 省電和其他瀏覽器設置以限制Flash使用不再對玩家有任何影響不論玩家的身高

此版本中的其他功能包括：

  1. 新皮膚 -根據重要的UX研究，我們構建了一個包含許多更改的新外觀。其中包括一個新的居中大播放按鈕，新顏色，新搜索欄行為以及一個新的標題欄（稱為停靠欄）。可用性測試參與者特別喜歡新的時間表，該時間表在懸停時有所擴展，使查找特定時間變得更加容易。現在，您可以在默認的新皮膚（稱為Luna）和舊的1.x皮膚（現在稱為Graphite）之間進行選擇。
  2. 初步支持實時字幕 -此版本現在支持許多608/708字幕編碼系統。這包括對流行的彈出和匯總方法以及基本樣式的支持
  3. 兼容模式適用於所有1.x和Video.js 4.x插件。 （DASH / DRM，FreeWheel和OnceUX插件除外-在第一季度將進行更新）

同樣在此版本中，我們進行了許多更新，以改進用於自定義的編程模型，包括簡化了外觀模型和插件API。這些更改為我們帶來了最新的開源版本Video.js 5的最新信息。您還將注意到，我們還在Video.js和Brightcove Player版本之間同步了版本編號。
