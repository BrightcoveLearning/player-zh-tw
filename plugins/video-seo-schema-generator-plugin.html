安裝插件

該插件的安裝方式與任何自定義插件一樣。以下是使用Studio和代碼添加插件的說明。

使用PLAYERS模塊安裝

  1. 打開球員模塊並創建一個新播放器或找到您希望添加插件的播放器。
  2. 單擊播放器的鏈接以打開播放器的屬性。
  3. 點擊插件在左側導航菜單中。
  4. 來自添加插件下拉菜單，選擇自定義插件。
  5. 為了插件名稱進入schema。
  6. 在這個網址中，輸入： 
    https://cdn.jsdelivr.net/npm/videojs-schema/dist/videojs-schema.min.js
  7. 不的CSS URL是必需的。
  8. 您無需在此特定實現中為此插件輸入任何選項。
  9. 點擊保存。
  10. 若要發佈播放程式，請按一下「發佈與內嵌」>「發佈變更」。
  11. 若要關閉開啟對話方塊，請按一下「關閉」。

使用代碼安裝

如果您希望使用代碼為單個播放器安裝插件，請使用以下代碼作為指南：

<video id="myPlayerID"
data-video-id="5796745147001"
data-account="1507807800001"
data-player="default"
data-embed="default"
data-application-id
class="video-js"
controls
width="640"
height="360"></video>
<script src="//players.brightcove.net/1507807800001/default_default/index.min.js"></script>

<script src="https://cdn.jsdelivr.net/npm/videojs-schema/dist/videojs-schema.min.js"></script>

<script>
videojs.getPlayer('myPlayerID').ready(function() {
var myPlayer = this;
myPlayer.schema();
});
</script>

使用的架構

該插件用於將Video Cloud視頻中的元數據注入到頁面中，如下所示： Schema.org結構化數據。它使用JSON-LD格式Google支持的視頻SEO：

<script type="application/ld+json">
{
"@context":"http://schema.org/",
"@type":"VideoObject",
"name":"Big Buck Bunny",
"description":"(c) copyright 2008, Blender Foundation / www.bigbuckbunny.org",
"thumbnailUrl":"https://cf-images.eu-west-1.prod.boltdns.net/v1/static/906043040001/f7d56300-ffca-460d-8ff6-fef835c12b36/e284fdba-d15a-422f-bebc-6a355e3d4dd3/1280x720/match/image.jpg",
"uploadDate":"2016-10-20",
"@id":"http://players.brightcove.net/906043040001/5WnZb7ptr_default/index.html?videoId=ref:bunnyfull",
"duration":"PT10M35S",
"embedUrl":"http://players.brightcove.net/906043040001/5WnZb7ptr_default/index.html?videoId=1401169490001"
}
</script>

插件選項

您可以在此插件中使用六個選項：

  • schemaId：用作的值@id在元數據中。這必須是代表視頻的唯一URI，但不一定是“真實" URL。這是任意的，可能是類似以下的值：
    • https://mydomain.com/videos/
    • https://videos.mydomain.com#
    • https://mydomain.com/article-about-video- {id}＃該視頻

    接受以下宏作為宏：

    • {id}
    • {referenceId}
    • {playerId}
    • {embedId}
    • {accountId}

    默認URI為：

    https://players.brightcove.net/{accountId}/{playerId}_{embedId}/index.html?videoId={id}
  • keywords：如果真的，包括標籤作為關鍵字；默認是錯誤的。
  • excludeTags：不包含為關鍵字的標籤數組，例如[“youtubesync"]。
  • baseObject：一個選項將視頻專用元數據構建到的屬性的對象。例如，可以使用它包括一個發布者對象： 
    "baseObject": {
  "publisher": {
    "@type": "Organization",
    "name": "Publisher name",
    "logo": {
      "@type": "ImageObject",
      "url": "https://example.com/logo.jpg",
      "width": 600,
      "height": 60
    }
  }
}
  • includeEmbedUrl：如果錯誤的，不包含嵌入網址。搜索引擎可能希望包含此URL。默認為真的。
  • preferLongDescription：如果真的，請使用詳細描述（如果有）。如果錯誤的， 或者真的並且未設置長描述，則將使用（簡短）描述字段。默認為錯誤的。
