例子

假設您有一個上面裝有Brightcove Player的網頁。現在，您想讓用戶跨社交媒體平台共享視頻。使用社交媒體插件，您將在標題和說明停靠欄中看到一個社交共享圖標。

注意：如果播放器配置為不顯示標題和描述停靠欄，則共享圖標將出現在播放器控制區域中。

單擊此圖標將顯示社交共享屏幕，如下所示。

您需要為播放器設置寬度和高度，這有兩個原因：

  1. 如果未設置寬度或高度，將沒有任何值傳遞給Twitter，這將導致Twitter卡無法驗證。
  2. 寬度需要足夠寬以容納社交插件的界面，如上面的屏幕快照所示。您也可以使用displayAfterVideo 屬性，記錄在下面。

注意：在Studio中，有一個選項可以向您的播放器添加社交分享，並且在許多情況下足以滿足您的需求。本文檔涵蓋了使用社交媒體插件的所有方面。以下屏幕截圖顯示了Studio的用戶界面。

點擊這裡用於使用該插件並包含社交媒體元標記的網頁的工作示例。

概述

本文檔將引導您逐步完成啟用Brightcove Player的現有HTML頁面的社交媒體共享的步驟。假設您已經使用正確的配置信息創建了Brightcove播放器，並且已將播放器添加到HTML頁面。

要添加社交媒體插件，請按照以下步驟操作：

  1. 配置播放器。這將更新Brightcove Player以使用社交媒體插件。與玩家分享社交內容的方式有以下三種：

    • 配置播放器屬性
      在工作室的球員模塊，你可以在裡面配置屬性社會的部分 .這提供了選擇不同社交媒體平台的界面，但沒有提供其他配置選項。

    • 使用選項配置播放器
      在工作室的球員模塊使用插件部分，有一個界面提供配置選項。

    • 自定義共享 URL
      此實現使用代碼添加社交插件。

  2. 新增社交元標記播放器的HTML頁面。
  3. 證實 HTML頁面上用於FaceBook和Twitter的meta標籤。

在瀏覽器中，使用Brightcove Player打開HTML頁面，然後使用播放器中的社交共享圖標共享您的視頻。

您也可以在頁面代碼中實現社交媒體插件，但這是一個更高級的解決方案。您可能要使用此方法來包含自定義共享URL。有關詳細信息，請參見自定義共享URL部分。

用於發佈到社交平台的其他選項包括：

  • 在Facebook即時文章中發布Brightcove Player
  • 將視頻發佈到社交媒體平台

配置播放器

要添加社交共享插件，請按照以下步驟操作：

  1. 打開球員模塊並創建一個新播放器或找到您希望添加插件的播放器。
  2. 選取播放程式的連結，以開啟播放程式的內容。
  3. 選擇社會的在左側導航菜單中。
  4. 接下來，選中復選框啟用分享。
  5. 選擇您要共享視頻的社交媒體平台。
  6. 若要發佈播放程式，請選取「發佈和嵌入」>「發佈變更」。
  7. 若要關閉開啟的對話方塊，請選取「關閉」。

使用選項配置播放器

對於更高級的配置，您可以使用特性作為配置插件的選項。為此，請按照下列步驟操作：

  1. 打開球員模塊並創建一個新播放器或找到您希望添加插件的播放器。
  2. 選取播放程式的連結，以開啟播放程式的內容。
  3. 選擇社會的在左側導航菜單中，並確保未選中該複選框。
  4. 選擇插件在左側導航菜單中。
  5. 來自添加插件下拉菜單，選擇自定義插件。
  6. 為了插件名稱進入social。
  7. 在這個網址中，輸入： 
    https://players.brightcove.net/videojs-social/3/videojs-social.min.js
  8. 為了的CSS URL，輸入： 
    https://players.brightcove.net/videojs-social/3/videojs-social.css
  9. 在選項（JSON）文本框。在此示例中，選項執行以下操作：
    • 第2-3行：設置視頻的名稱和描述。
    • 第4行：使用視頻播放器和社交共享元標記定義自定義網頁的URL。
    • 第5-6行：對於Twitter，這允許用戶在5秒鐘的偏移後開始觀看視頻。

    • 第7-13行：啟用Facebook，Twitter，Tumblr，Pinterest和LinkedIn的社交共享。

    {
  "title": "Tiger",
  "description": "Video of two tigers",
  "url": "https://solutions.brightcove.com/bcls/brightcove-player/social-plugin/social-media.html",
  "deeplinking": true,
  "offset": "00:00:05",
  "services": {
    "facebook": true,
    "twitter": true,
    "tumblr": true,
    "pinterest": true,
    "linkedin": true
  }
}
  10. 點擊保存。
  11. 若要發佈播放程式，請按一下「發佈與內嵌」>「發佈變更」。
  12. 若要關閉開啟對話方塊，請按一下「關閉」。

性質

對於一般的插件，options 對像用於將數據傳遞到插件以自定義初始化。在這種情況下，可以在此對像中使用以下屬性：

標題

  • 類型：字符串
  • 這是一個自定義標題，將在您共享視頻時顯示。

描述

  • 類型：字符串
  • 這是一個自定義描述，將由支持它的社會服務使用。

網址

  • 類型：字符串
  • 這是指向自定義網頁的URL，該自定義網頁包含您的視頻和要共享的meta標籤。請參閱元標記部分，了解有關如何將社交媒體元數據添加到播放器頁面的詳細信息。

嵌入代碼

  • 類型：字符串
  • 這是Brightcove Player iframe嵌入代碼，用於共享視頻。這樣您就可以完全覆蓋嵌入代碼位於共享對話框中的字段。
  • 您可以使用getEmbedCode（）方法。

removeEmbed

  • 類型：布爾值
  • 默認：false
  • 如果設置為true，關閉（隱藏）iframe嵌入代碼

displayAfterVideo

  • 類型：布爾值
  • 默認：false
  • 使社交屏幕自動顯示在ended 事件

深層連結

  • 類型：布爾值
  • 默認：false

  • 這將啟用/禁用共享視頻的深層鏈接。此功能允許用戶從特定的偏移量開始觀看視頻。有關更多詳細信息，請查看深層連結文檔。

    注意：深層鏈接功能目前僅適用於 Twitter。

抵消

  • 類型：字符串
  • 格式：00h00m00s
  • 這與deeplinking 屬性，並定義何時開始播放視頻。要抵消1分5秒，您可以這樣寫：00h01m05s

服務

在此對像中包括所有服務屬性。要啟用或禁用對服務的支持，請將屬性值設置為true或false而不是刪除它們。

  • facebook
    • 類型：布爾值
    • 默認：true
    • 這將啟用Facebook共享圖標。
  • twitter
    • 類型：布爾值
    • 默認：true
    • 這將啟用Twitter共享圖標。
  • tumblr
    • 類型：布爾值
    • 默認：true
    • 這將啟用Tumblr共享圖標。
  • pinterest
    • 類型：布爾值
    • 默認：true
    • 這將啟用Pinterest共享圖標。
  • linkedin
    • 類型：布爾值
    • 默認：true
    • 這將啟用LinkedIn共享圖標。

buttonParent

  • 類型：文字
  • 默認：shelf
  • 確定共享圖標應位於的位置。如果設置為架子，即默認設置，共享圖標將顯示在擴展塢中。如果該值設置為控制欄，共享圖標將出現在控制欄中。

例子：

{
  "services": {
    "linkedin": true,
    "pinterest": true,
    "tumblr": true,
    "twitter": true,
    "facebook": true
  },
  "offset": "00:00:05",
  "deeplinking": true,
  "url": "https://solutions.brightcove.com/bcls/brightcove-player/social-plugin/social-media.html",
  "description": "Video of two tigers",
  "buttonParent": "controlBar",
  "title": "Tiger"
}

Getter和Setter配置方法

有許多方法可用於設置和檢索配置選項。所有方法都與社會覆蓋組件，擴展了videojs.ModalDialog班級。（有關該類的信息，請參見實現ModalDialog文檔。）

初始化社交插件後，社會覆蓋組件在以下位置可用player.socialOverlay 。使用getter和setter方法的一般格式如下：

player.socialOverlay.methodName()

setDirectLink（字符串）

  • 用法：player.socialOverlay.setDirectLink(string)
  • 目的：更新社交覆蓋層使用的URL。將值設置為空字符串（''）將導致根據需要確定URL。

getDirectLink（）

  • 用法：player.socialOverlay.getDirectLink()
  • 目的：返回社交疊加層使用的URL。

setEmbedCode（字符串）

  • 用法：player.socialOverlay.setEmbedCode(string)
  • 目的：更新社交疊加層使用的嵌入代碼。

getEmbedCode（）

  • 用法：player.socialOverlay.getEmbedCode()
  • 目的：獲取社交疊加層使用的嵌入代碼。

社交按鈕

初始化社交插件後，社交按鈕組件在以下位置可用player.socialButton 。該組件擴展了videojs.Button班級。

在臉書上分享

目前，只有Facebook的本地播放器才能在Facebook牆中播放視頻。如果您想共享視頻並在Facebook牆上播放這些視頻，則可以使用Brightcove Social。有關詳細信息，請參見Brightcove Social入門文檔。

Brightcove Player將在牆上創建一個鏈接，該鏈接將在新的瀏覽器選項卡或窗口中打開您的視頻和播放器。對於大多數人來說，這種行為可能很好。為此，Facebook允許您使用Open Graph（ OG）頁面HTML頁面中的meta標籤。有關如何使用的詳細信息OG Facebook標籤，請參閱分享最佳實踐文檔。

要共享播放器和視頻的鏈接，請按照以下步驟操作：

  1. 在播放器模塊中，向下滾動到社交分享部分，然後選擇是的允許共享。
  2. 保存您的更改並發布您的播放器。
  3. 使用播放器和視頻創建一個HTML頁面。
  4. 產生您的社交元標籤為您的HTML頁面進行更新，如下所示：
    • 第9-11行：定義您的網站名稱，視頻標題，然後輸入視頻。

    • 第13行：指向視頻的縮略圖。

在Twitter上分享

定制服務

該插件內置了對六個流行社交媒體平台的支持，但您也可以使用自定義服務添加其他插件。為此，您需要配置一個options.customServices 目的。該對像是自定義服務對象的數組。顯示了一個簡短的示例配置，然後提供了示例的詳細信息。

以下示例將添加三個按鈕。Xing和WhatsApp的第一個和第二個按鈕。WhatsApp按鈕將僅出現在移動設備上。這兩個示例都在圖標上使用了Font Awesome，如果播放器是已嵌入到已經使用該字體的頁面中的頁面內播放器，則可以方便使用，但是可以使用任何其他樣式。第三個是使用Facebook的feed API的Facebook共享按鈕，該按鈕允許包含Facebook應用ID進行歸因。使用此功能時，您想禁用常規的Facebook按鈕。

[{
  className: 'fa fa-xing',
  title: 'Xing',
  hrefTemplate: 'https://www.xing.com/spi/shares/new?url={{url}}',
  bgColor: '#026466'
},
{
  className: 'fa fa-whatsapp',
  title: 'WhatsApp',
  hrefTemplate: 'https://api.whatsapp.com/send?phone=&text=title%20{{url}}',
  bgColor: '#25d366',
  mobileOnly: true
},
{
  classSuffix: 'facebook',
  title: 'Facebook',
  hrefTemplate: 'https://www.facebook.com/dialog/feed?app_id=123456789&caption={{title}}&display=page&link={{url}}'
}]

以下定義了定制服務對象的屬性：

customService.title

  • 類型：字符串
  • 共享目的地的標題。

customService.className

  • 類型：字符串
  • 一個要添加到按鈕的類或類的列表（以空格分隔）。

customService.classSuffix

  • 類型：字符串
  • 後綴附加到vjs-icon- 生成按鈕的類。僅在以下情況下使用className 未指定。僅在替換標準目標位置之一時才有用，因為Video.js字體不包含其他社交目標圖標。

customService.hrefTemplate

  • 類型：字符串
  • 用於生成共享URL的字符串模板，其中可能包含宏{{url}} ，{{poster}} ，{{title}} 和{{description}} 。可以代替customService.href 無法指定功能的地方，例如，將JSON字符串中的選項提供給Player Management API。

customService.href

  • 類型：字符串
  • 生成共享URL的功能。必須帶一個對象並返回一個字符串。除非特別說明，否則這是從hrefTemplate 。

customService.bgColor

  • 類型：字符串
  • 可選的CSS顏色值，將用作背景顏色元素樣式。如果按鈕由CSS設置樣式，則不需要此按鈕，但為方便起見，特別是在圖標將由現有樣式表設置樣式的情況下提供。

customService.mobileOnly

  • 類型：布爾值
  • 默認值：false
  • 如果true ，該按鈕將僅在Android和iOS上顯示。

已知的問題

終端屏幕上沒有簡短說明

影片的簡短的介紹如果在播放視頻時單擊共享按鈕，則會顯示。在包含共享信息的最終屏幕上，簡短的介紹不顯示。這是按設計的。

按鈕的鍵盤導航-適用於OS X的Firefox

Firefox for OSX具有默認的製表符行為，可僅在文本框中製表並僅列出元素。由於“社交插件"的按鈕是鏈接元素，因此除非您選中此框，否則無法使用鍵盤瀏覽它們所有控件在下面系統偏好設置>鍵盤>快捷方式。
