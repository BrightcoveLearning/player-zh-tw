概述

Tealium插件可讓您將Tealium iQ標籤管理器與Brightcove Player集成。這樣，您就可以使用Tealium Universal Data Hub跟踪Brightcove Player事件。

標記管理器允許您添加和發布標記，而不必使用代碼。因為它們是客戶端解決方案，所以標籤管理器使您可以跨不同的營銷技術收集數據。這使營銷人員能夠跨各種收集點收集用戶和播放器數據，從而為他們提供支持。

由於Tealium iQ不是為特定平台設計的，因此可以將其與您選擇的產品一起使用。使用Tealium iQ，您可以訪問標籤市場，在那裡您可以找到滿足大多數需求的標籤模板。

Brightcove Player 版本 6.36.5+ 支持 Tealium 標籤管理器插件。

標籤管理術語

以下是使用Tealium iQ時會遇到的一些常見術語。

學期 描述
事件 事件是用於將數據發送到報告工具的數據點。這些與視頻加載和播放有關。
輪廓 您的標籤管理系統帳戶可以有多個容器（配置文件），並具有自己的標籤和變量。每個概要文件可以具有多個環境，通常是為開發，登台和生產而設置的。
規則 規則告訴您何時應將標記代碼添加到頁面。這些是用戶定義的條件。
標籤 標籤是放置在您的網頁上的代碼片段，用於收集用戶交互數據。標記管理系統簡化了將標記添加到站點的過程，僅要求您提供標記ID和標記事件類型。
標籤市場 標籤市場模板提供用於部署標籤的代碼段。儘管這減少了所需的代碼知識量，但要成功實現還需要一定程度的對JavaScript和HTML的熟悉。
追踪器 跟踪器是用於收集，存儲數據並將該數據發送到Google跟踪代碼管理器的對象。
通用數據對象（UDO）utag_data對象 一個JavaScript變量，稱為utag_data其中包含駐留在您網頁代碼中的數據集合。此JavaScript對像用於將信息從您的網站傳遞到標籤管理器容器。然後，您可以使用此信息來填充變量並激活標籤配置中的事件。

入門

查看Tealium iQ標籤管理文檔來設置您的帳戶。該設置將包括以下內容：

  • 定義標籤配置
  • 定義觸發信標的觸發器

  • 為事件和自定義維度創建變量

使用Brightcove插件

與其他Brightcove Player插件一樣，必須將Tealium插件加載到播放器中。

使用工作室

以下步驟詳細說明了使用Video Cloud Studio加載插件。

  1. 打開球員模塊並創建一個新播放器或找到您希望添加插件的播放器。
  2. 選取播放程式的連結，以開啟播放程式的內容。
  3. 選擇插件在左側導航菜單中。

  4. 接下來，選取 [ 新增外掛程式] 按鈕，然後選取 [ Brightcove 外掛程式] 。

  5. 展開Brightcove插件下拉菜單，然後選擇茶葉。

  6. 現在，您就可以使用要跟踪的維度來配置Google Analytics（分析）了。查看可用的尺寸，請參閱事件和屬性本文檔的第二部分。

  7. 在裡面選項（JSON）文本框，輸入您要跟踪的事件和屬性。欄位說明：

    • paramsToTrack - List of attributes you want to track
    • eventsToTrack - List of events you want to track
    • account - Tealium account name
    • profile - Tealium profile name
    • env - Tealium environment name
     

    這是一個例子：

    {
  "paramsToTrack": {
    "video_playhead": "video_playhead",
    "video_platform": "video_platform",
    "video_length": "video_length",
    "video_name": "video_name",
    "video_id": "video_id",
    "video_milestone": "video_milestone"
  },
  "eventsToTrack": {
    "video_load": "video_load",
    "video_play": "video_play",
    "video_complete": "video_complete",
    "video_watch": "video_watch",
    "video_pause": "video_pause",
    "ad_start": "ad_start",
    "ad_end": "ad_end"
  },
  "env": "your environment",
  "profile": "your profile",
  "account": "your account"
}

  8. 選取「儲存」按鈕。您現在將看到茶葉插件已添加到播放器的插件列表中。

  9. 若要發佈播放程式，請選取「發佈和嵌入」>「發佈變更」。
  10. 要關閉對話框，請選擇關閉。

  11. 返回到媒體並使用剛為Tealium更新的播放器發布視頻或播放列表。

    當播放器加載並且用戶與播放器交互時，將跟踪指定的事件和參數。

事件和屬性

以下事件和屬性可發送到Tealium iQ。

活動

這些事件來自您的播放器。所有這些事件都發送公共屬性。某些事件可能會發送其他屬性。

事件 類型 描述
video_load 標準 視頻元數據在播放器中加載完畢後。
video_play 標準 當用戶或自動播放啟動播放時
已傳送其他屬性：
  • video_playhead
video_watch [1] 風俗 這包括附加的演繹選擇，比特率測量和緩衝信息。
已傳送其他屬性：
  • video_playhead
  • video_milestone
video_pause 標準 當用戶暫停視頻播放時
已傳送其他屬性：
  • video_playhead
ad_start 風俗 廣告播放開始後
ad_end 風俗 廣告播放結束後
video_complete 標準 視頻播放結束時觸發。
筆記

以下注意事項是指上表。

[ 1] 這video_watch事件在以下里程碑發送：

  • 25％里程碑–觀看視頻的時間佔視頻總時長的25％至50％
  • 50％里程碑-觀看視頻的時間佔視頻總時長的50％至75％
  • 75％的里程碑-觀看視頻的時間佔視頻總時長的75％至95％
  • 100％里程碑-觀看視頻時長為視頻總時長的100％

觀看視頻的百分比（video_watch）的計算方式如下（類似於Google和Adobe Analytics中的video_engagement事件）：

Video watched percentage = duration the user actually watched / total video duration

例如，在20秒的視頻中，以下兩種情況均可實現25％的里程碑

  • 用戶觀看時間為0-5秒
  • 用戶從0-1秒觀看，然後跳到11秒，然後觀看15秒

在上述兩種情況下，用戶觀看了5秒，因此里程碑將報告為25％。

共同屬性

這些屬性在所有事件中都是通用的。

名稱 類型 描述
video_id 字串 發起事件的Brightcove視頻ID
video_length 編號 video屬性指定的視頻長度
video_name 字串 video屬性指定的視頻名稱
video_platform 字串 正在運行視頻的平台

特定於事件的屬性

這些屬性與特定事件有關。

名稱 類型 描述
video_playhead 編號 視頻中觸發事件的時間。
隨事件一起發送：
  • video_play
  • video_pause
  • video_watch
video_milestone 編號 觸發事件時完成的視頻的百分比。
隨事件一起發送：
  • video_watch

在Tealium中顯示數據

玩家動作觸發插件將分析數據發送到您的Tealium通用數據中心。要查看事件和屬性，請按照下列步驟操作：

  1. 去https://my.tealiumiq.com
  2. 使用您的憑據登錄到通用數據中心。
  3. 在左側導航欄中，展開事件流。

  4. 在左側導航中，選擇現場活動。

  5. 您應該看到帶有實時事件數據的儀表板。

  6. 選擇一個事件以查看與該事件關聯的屬性。
