iOS和Brightcove Player

{{ page.description }}

自動播放視頻

鑑於不必擔心可能會下載不需要的片段，您可以自動播放視頻。為此，您必須執行以下操作：

  • 添加autoplay屬性。對於iframe嵌入代碼，將此屬性設置為true。
  • 添加playsinline屬性。對於iframe嵌入代碼，將此屬性設置為true。
  • 添加muted屬性。對於iframe嵌入代碼，將此屬性設置為true。或者，您可以使用沒有音軌的視頻。

頁內嵌入代碼的實現：

<video-js data-video-id="5045678908001"
	data-account="1752604059001"
	data-player="default"
	data-embed="default"
	data-application-id
	class="video-js"
	autoplay playsinline muted
	controls></video-js>
<script src="https://players.brightcove.net/1752604059001/default_default/index.min.js"></script>

iframe嵌入代碼的實現：

<iframe src="https://players.brightcove.net/1752604059001/default_default/index.html?
  videoId=5811862231001&autoplay=true&muted=true&playsinline=true"
	allowfullscreen
	webkitallowfullscreen
	mozallowfullscreen></iframe>

有關自動播放的完整討論，請參見自動播放注意事項文檔。

非全屏播放視頻

要非全屏播放視頻，您必須執行以下步驟：

  • 設置playsinline歸因於true。

在iOS 10及更高版本中，playsinline屬性允許Brightcove Player在不強制全屏的情況下開始播放。觀看者可以播放嵌入式視頻，但不會自動進入全屏模式。這是iOS的默認行為。

當觀看者開始視頻播放然後進入全屏顯示時，視頻播放不會中斷。

當用戶退出全屏模式時，視頻播放將暫停。這是 iOS 限制。

全屏播放視頻

當用戶從嵌入式播放切換到全屏模式時，將使用iOS本機播放器而不是Brightcove Player，但以下一項除外：

  • 裝有Safari 12的iPad在iOS 12上運行，它將自動運行Brightcove Player。

使用本機播放器可使播放器充分利用整個設備的屏幕。顯然，刪除Brightcove Player控件後，某些功能可能會丟失。如果您不想只允許iOS設備使用全屏顯示，則可以使用以下代碼：

<script>
  videojs.getPlayer('myPlayerID').ready(function() {
    var myPlayer = this,
		fullScreenElement = document.getElementsByClassName("vjs-fullscreen-control")[0];
	if (videojs.browser.IS_IOS ) {
		fullScreenElement.parentNode.removeChild(fullScreenElement);
	}
  });
</script>

停止不必要的片段下載

iOS可能會下載不需要的段，從而導致帶寬佔用過多。在這種情況下autoplay啟用後，即使preload設定為none。防止這種情況的唯一方法是不使用autoplay屬性。

筆記：

  • 有人誤解了preload歸因於none將阻止下載細分。這不是真的。
  • 使用伺服器端廣告插入 (SSAI) 時，Brightcove 建議設定preload=none。如果preload被設定為auto或者metadata，將請求視頻流的第一幀，即使用戶沒有開始播放，這也會將廣告計為已投放。
