介紹

Brightcove為視頻清單庫中的資產的視頻清單文件提供靜態URL。這使您可以靈活地在自己的 CMS 中管理內容，並使用自定義安全架構進行交付。

這對於現有架構不允許在需要清單 url 之前調用 Playback API 的客戶很重要。播放器也可以使用此功能，通過消除一個呼叫來減少播放開始時間。

另請參閱播放API v2參考。

需求

靜態 URL 需要以下要求：

  • Brightcove播放API v2
  • 您的資產被提取用於動態交付

分析註冊

使用靜態 URL，您可以在不使用 Brightcove Player 的情況下與 Playback API 集成。雖然 Brightcove Player 會自動將分析數據發送到 Brightcove，但您將使用數據收集 API 來執行此操作。

如果您不熟悉Data Collection API，請查看概述：資料收集API文檔。

有關將信息發送到數據收集器的詳細信息，請參見數據收集API樣本。

視頻雲

今天，每次通過 VideoCloud 播放都需要調用 2 個 API 來開始視頻播放。通過使用靜態 URL，您可以將 API 調用次數減少到一次。

實施靜態網址

靜態 URL 包括以下內容：

  • 帳戶ID
  • 視頻 ID（或參考）
  • 投放類型（HLS，破折號）
  • 配置選項（SSAI，DRM，CDN等）

靜態 URL 使用客戶生成和簽名的JSON 網絡令牌 (JWT)。使用此令牌，您可以配置和實施交付的各個方面。有關詳細信息，請參閱創建一個JSON 網絡令牌部分。

領域

用於靜態 URL 的域是 Playback API v2 (PAPIv2) URL： 

https://playback.brightcovecdn.com/playback/v1/

創建一個JSON 網絡令牌

要創建在與 Brightcove 的 Playback API 通信時使用的令牌，請參閱創建一個JSON 網絡令牌 (智威湯遜)文檔。您會在那裡找到可用於靜態 URL 傳遞的聲明。

使用內容保護 (DRM)

您可以包括pro聲明，如上定義，指定保護類型。

對於許可證請求，JWT不需要任何有效負載，但是必須使用與清單請求JWT相同的RSA密鑰對其進行簽名。

Widevine 許可證請求

您可以使用以下端點獲取 Widevine 的 DRM 許可證： 

/accounts/{account_id}/videos/{video_id}/licenses/cenc/widevine?bcov_auth={JWT}

FairPlay應用程序證書

對於FairPlay應用程序證書，端點為： 

/accounts/{account_id}/fairplay_app_cert

PlayReady端點

對於PlayReady，許可請求的端點為：

/accounts/{account_id}/videos/{video_id}/licenses/cenc/playready?bcov_auth={JWT}

筆記

以下注意事項適用於靜態 URL 傳遞：

  • 支持清單內字幕。將來的版本將支持帶外Sidecar字幕。
  • 客戶的 BYO CDN 將提供視頻片段，而 Brightcove 的 CDN 將繼續提供清單和子清單。
  • Brightcove支持使用永不過期的JWT。為此，請排除exp創建令牌時聲明。
