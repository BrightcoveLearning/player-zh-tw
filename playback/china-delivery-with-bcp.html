介紹

Brightcove提供了從國外向中國交付視頻的解決方案。解決方案由兩部分組成：

  • 中國境內託管的代理服務器
  • 關聯的中國特定CDN

代理將非視頻流量（播放 API 請求、分析信標和網絡播放器本身）路由到中國觀眾和來自外部的 Brightcove 基礎設施。

CDN 連接到代理並處理視頻和圖像內容。

配置Brightcove Player

要在中國境內交付視頻，您必須配置播放器以使用Brightcove China交付代理。

要創建與中國兼容的播放器，請按照以下步驟操作：

  1. 在Video Cloud Studio中，打開選手們模塊。使用現有播放器或創建一個新播放器。

    請記住使用 Brightcove Player 6.44 或更新版本（推薦最新版本）。
  2. 選擇播放器的名稱鏈接以打開播放器的屬性。

  3. 在左側導航中，選擇JSON編輯器。

  4. 要設置代理，請添加以下內容：

    "china_proxy_domain": "china.example.com"
    這應該只是一個主機名。沒有協議（例如，https://），也沒有包括正斜杠在內的特殊字符。

  6. 點擊節省，並發布播放器。
  7. 當您嵌入或共享您的播放器時，請注意嵌入代碼使用 Brightcove Player CDN。 
    <script
  src="//players.brightcove.net/1752604059001/jnOD8yLME_default/index.min.js">
</script>
  8. 更改 Brightcove Player CDN 以使用您的 China Delivery 域。 
    <script
  src="//china.example.com/1752604059001/jnOD8yLME_default/index.min.js">
</script>
    使用players.brightcove.net域將導致在中國無法播放內容和降低播放器加載時間。

筆記

和china_proxy_domain設置後，播放器發起的所有請求都將發送到該域內的URL。使用上面的樣本值（ china.example.com）用戶可能會看到以下要求：

系統 小路
播放器CDN //china.example.com/players / ...
播放接口 //china.example.com/playback/v1 / ...
分析 //china.example.com/v2/tracker ...

本土化

和china_proxy_domain設置，Brightcove Player將自動地配置兩種語言：

  • zh-Hans：簡體中文（默認）
  • zh-Hant：繁體中文

設置一個china_proxy_domain與設置以下配置具有相同的效果（儘管不需要這些設置）：

"language": "zh-Hans",
"languages": ["zh-Hans", "zh-Hant"],

此默認效果china_proxy_domain如果為language或者languages配置！要覆蓋默認行為，您可以混合併匹配這些配置。

添加更多語言

如果語言多於zh-Hans和zh-Hant需要時，您可以向播放器添加語言配置。此配置可能類似於：

"china_proxy_domain": "china.example.com",
"languages": ["zh-Hans", "zh-Hant", "en", "jp"],

在此示例中，播放器仍默認為zh-Hans，但還會添加英語和日語。

更改默認語言

要配置與默認語言不同的默認語言zh-Hans，添加一個language配置。該配置應類似於以下內容：

"china_proxy_domain": "china.example.com",
"language": "zh-Hant",
