玩家目錄

在本主題中，您將學習使用Brightcove Player的目錄向Video Cloud目錄發出請求的方法庫。該目錄的庫可輕鬆獲取有關Video Cloud媒體的信息並將其加載到播放器中。

本文件中的內容僅適用於視訊雲端。

基本語法

使用的基本語法catalog如下：

  • 方法：playerName.catalog.methodName()

幾個catalog方法將傳遞定義請求的參數對象。這目錄參數對象在下一節中定義，並將在方法中引用說明（如適用）。

注意：在本頁的所有內容中，當參數被方括號 ( [ ] ) 表示該參數是可選的。

目錄參數對象

下表描述了CatalogParams目的。這個真實的對象（不是構造函數）定義了用於許多目錄請求的參數格式。

名稱 類型 描述
type string

發出請求的類型。必須是以下之一視頻，播放清單， 或者搜索。
policyKey string

此帳戶的策略密鑰。這將默認為播放器的策略鍵。 **請注意，如果您將不同的帳戶 ID 作為參數傳遞，則還必須為該帳戶傳遞相應的策略密鑰。
deliveryConfigId string

投放規則ID。看概述：交貨規則。
[id] string

以“ref:”為前綴的視頻或播放列表 ID 或參考 ID。必需的用於視頻和播放列表請求。棄用：為了向後兼容，這被支持作為搜索查詢而不是q .可以將其作為子對象提供，它也是一個CatalogParams對象，可用於提供參數。
[q] string

一個搜索查詢。必需的對於搜索請求，其他人忽略。
[adConfigId] string

視頻可以SSAI廣告配置ID。
[tveToken] string

可選的TVE令牌，將作為查詢字符串參數發送。
[limit] string

僅支援播放清單和搜尋類型。 限制返回的視頻數量。
[offset] string

僅支援播放清單和搜尋類型。 要跳過的視頻數。
[sort] string

支持搜索僅鍵入。 視頻應如何分類以進行搜索。
[bcovAuthToken] string 與Brightcove播放授權服務（PAS）一起使用。

**筆記：默認情況下，不啟用播放器生成的策略密鑰。這意味著您必須（選擇一個）：

  • 使用policyKey參數以傳遞啟用搜索的策略密鑰。

  • 使用播放器管理API覆蓋播放器的默認策略密鑰。您可以使用類似於以下內容的curl語句來這樣做：

    curl \
  --header "Content-Type: application/json" \
  --user $EMAIL \
  --request PATCH \
  --data '{
    "video_cloud": {
      "policy_key": "YOUR SEARCH ENABLED POLICY KEY"
    }
  }' \
https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players/$PLAYER_ID/configuration

    將啟用搜索的策略密鑰與播放器相關聯時，請務必考慮任何安全問題。

    有關創建啟用搜索的策略密鑰的信息，請參見概述：政策API-搜索視頻部分。

getVideo( ) method

這getVideo(params | videoID,callback,[adConfigId]) => XMLHttpRequest方法使catalog請求具有指定ID的視頻，並在請求完成後調用回調。

參數

  • 參數 |視頻ID：類型對象目錄參數或包含視頻 ID 或參考 ID 前綴的字符串參考： .

  • 回呼：無論請求成功與否，都會在請求完成時調用該函數。它使用兩個參數來調用：

    • 如果發生錯誤，第一個參數將包含一個錯誤對象。該對象包含錯誤的詳細信息。如果沒有發生錯誤，則此物件為 null 。

    • 如果沒有錯誤發生，則第二個參數是請求的視頻對象。如果發生錯誤，則此值包含空字符串。

  • [ 管理員] ：已棄用視頻雲廣告配置ID，該ID將傳遞給Playback API。而是在初始params對像中傳遞adConfigId。

如果您希望加載播放列表，但沒有將播放列表中的第一個視頻放置在播放器中，您可以在該playlist()方法中將第二個參數設置為-1。例如，以下代碼檢索播放列表，將其放入播放器中，但不將第一個視頻加載到播放器中： 
myPlayer.catalog.getPlaylist(playlistID, function(error, myPlaylist) {
  // deal with error
  // +++ Place playlist in player, but -1 parameter does not load playlist video in player +++
  myPlayer.playlist(myPlaylist, -1);

當然，然後您必須手動加載視頻。見Brightcove Player示例：在播放清單前播放視訊，以取得此技術的範例。

get（）方法

這get(params, [callback]) ⇒ Promise | XMLHttpRequest方法允許搜索catalog使用CMS /播放API：影片搜尋句法。在後台，此方法使用Playback API。

參數

  • 參數 :類型目錄參數的物件。

  • 回呼：無論請求成功與否，都會在請求完成時調用該函數。它使用兩個參數來調用：

    • 如果發生錯誤，第一個參數將包含一個錯誤對象。該對象包含錯誤的詳細信息。如果沒有發生錯誤，則此物件為 null 。

    • 如果沒有錯誤發生，則第二個參數是請求的視頻對象。如果發生錯誤，則此值包含空字符串。

返回值

該方法返回一個承諾或類型的對象XMLHttpRequest .如果您使用的是回調函數，則將自動解析JSON響應，並且將有用的對像傳遞給回調函數。回調中返回的數據的確切結構將有所不同，具體取決於所搜索的內容。

回調示例

下面的示例執行以下操作：

  1. 構建一個catalogParams對象，該對象將作為該函數的參數傳遞。搜索將查詢帶有字符串的視頻沙漠在視頻的姓名 .

  2. 使用get()方法。請注意，回調函數將使用名為errorObj和videosReturned。

  3. 使用playlist()將返回的視頻分配給Brightcove Player的方法。

<script type="text/javascript">
  videojs.getPlayer('myPlayerID').ready(function() {
    var myPlayer = this,
      catalogParams = {};
    catalogParams.type = 'search';
    catalogParams.policyKey = 'BCpkADawqM3dv_...Cv8-nlTX';
    catalogParams.q = 'name:Desert';
    console.log('catalogParams',catalogParams);
    myPlayer.catalog.get(catalogParams,function(errorObj,videosReturned){
      console.log('videosReturned',videosReturned);
      myPlayer.playlist(videosReturned);
    });
  });
</script>

getSequence（）方法

這getSequence(sequences, callback, [adConfigId]) ⇒ Array.<XMLHttpRequest>方法允許多次通過catalog操作並從這些多個對像中檢索所有視頻對象catalog操作。您可以有三種類型的操作，它們是：

  1. 搜尋
  2. 視頻
  3. 播放清單

實際上，這意味著您可以檢索以下組合，並且返回的數據是視頻對象的數組：

  • 多個播放列表
  • 播放列表和其他一些單獨的視頻
  • 通過搜索，所有在名稱，描述或標籤中帶有特定字符串的視頻。

當然，剛剛列出的序列只是幾乎無限數量的組合中的幾個。

參數

  • 序列：類型對象目錄參數 OR 表示一系列對象的數組catalog操作。

  • 回呼：無論請求成功與否，都會在請求完成時調用該函數。它使用兩個參數來調用：

    • 如果發生錯誤，第一個參數將包含一個錯誤對象。該對象包含錯誤的詳細信息。如果沒有發生錯誤，則此物件為 null 。

    • 如果沒有錯誤發生，則第二個參數是請求的視頻對象。如果發生錯誤，則此值包含空字符串。

  • [ 管理員] ：與每個請求一起傳遞的廣告配置ID。

請注意，如果通過多個操作返回了特定視頻，則該視頻將多次出現在結果中。

返回值

該catalog調用本身返回類型的 XMLHTTP 任務的對象。雖然調用返回一個 XMLHttpRequest 對象，但你很可能永遠不會處理該對象。原因是在回調函數中，將自動解析JSON響應，並且將有用的對像傳遞給回調函數。回調中返回的數據的確切結構將有所不同，具體取決於所搜索的內容。

例子

以下示例使用三個序列catalog操作。首先是在catalog，第二次添加單個視頻，最後將播放列表的視頻添加到結果中。

<script>
  videojs.getPlayer('myPlayerID').ready(function() {
    var myPlayer = this,
      // Declare object for catalog search
      catalogParams = {};
    // Declare mySequences array which will hold
    // three different types of catalog operations
    mySequences = [];

    // Declare 3 objects for different catalog operations
    mySequences[0]={};
    mySequences[1]={};
    mySequences[2]={};

    // Define the object for catalog search
    // Search will retrieve 10 oldest videos in catalog
    catalogParams.policyKey = 'BCpkADawqM3dv_-rSSaDpwCVfj0vsWCv8-nlTX';
    catalogParams.q = 'type:*';
    catalogParams.sort = 'created_at';
    catalogParams.limit = 10;
    catalogParams.type = 'search';
    mySequences[0] = catalogParams;

    // Create a video retrieval catalog operation
    catalogParams = {};
    catalogParams.type = 'video';
    catalogParams.id = '5755775186001';
    mySequences[1] = catalogParams;

    // Create a playlist retrieval catalog operation
    catalogParams = {};
    catalogParams.type = 'playlist';
    catalogParams.id = '5531423971001';
    mySequences[2] = catalogParams;

    // Display mySequences array
    console.log('mySequences',mySequences);

    // Perform sequence of catalog operations
    myPlayer.catalog.getSequence(mySequences,function(errorObj,videosReturned){
      // Display video objects returned
      console.log('videosReturned',videosReturned);
      // Assign video objects as a playlist
      myPlayer.playlist(videosReturned);
    });
  });
</script>

getLazySequence（）方法

這getLazySequence(sequences, callback, [adConfigId]) ⇒ XMLHttpRequest方法的行為與getSequence（）方法，只是採用了延遲加載視頻。這意味著，當檢索到一個序列操作的視頻時，直到到達前一個操作的最後一個視頻時，才會檢索下一個序列操作的視頻。如果序列僅返回單個視頻，則存在特殊情況，在這種情況下，將執行下一個序列操作並檢索更多視頻。

autoFindAndLoadMedia（）方法

這autoFindAndLoadMedia([opts]) ⇒ Object | undefined方法查找最後定義的視頻或播放列表ID，然後從API進行請求，然後將其加載到播放器中。此方法定義Brightcove Player的默認行為。

參數

  • options：可以識別以下任何選項：

    • adConfigId (number):廣告配置的數字值，該數字值將傳遞到播放API。最後將稱為ad_config_id。

    • embedHasPoster (boolean):原始嵌入內容是否帶有海報。

    • embedHasSources (boolean):原始嵌入內容是否具有來源。

    • embedHasTracks (boolean):原始嵌入內容是否有曲目。

    • playlistId (string):播放器配置中提供的播放列表ID。

    • search (string | Object):播放器配置中提供或嵌入的搜索字符串/對象。看get()有關支持的參數的更多信息的方法。

    • sequence (Object | Array.Object):播放器配置中提供或嵌入的搜索字符串/對象。看get()有關支持的參數的更多信息的方法。

    • videoId (string):播放器配置中提供的視頻ID。

以下示例基於以下示例獲取視頻對象：videoId並將其加載到播放器中：

<script type="text/javascript">
  videojs.getPlayer('myPlayerID').ready(function() {

transformVideoResponse（）方法

這transformVideoResponse(data)方法從以下任一位置轉換原始視頻對象： CMS API或者播放API轉換為與播放器和瀏覽器兼容的格式。這包括持續時間和https圖片兼容性。它還重新安排了源，以確保首先嘗試在此平台上提供最高質量觀看體驗的源。

當使用catalog.getVideo()或者catalog.getPlaylist()方法，transformVideoResponse()被自動調用，以便您返回的數據採用適當的格式。

參數

  • data：從原始視頻對象解析的原始視頻對象catalog API響應。

 for (var i=0; i<limitValue; i++) {

目錄錯誤

這catalog檢索或最初播放視頻或播放列表時出現問題時，將引發錯誤。錯誤詳細說明如下。

目錄錯誤 描述
Video ID as string 播放器已配置了無效的Video Cloud帳戶ID。
VIDEO_CLOUD_ERR_RESOURCE_NOT_FOUND 默認播放列表ID無效。
VIDEO_CLOUD_ERR_VIDEO_NOT_FOUND 默認視頻無效。
VIDEO_CLOUD_ERR_NOT_PLAYABLE 視頻雲視頻無法播放。 導致此錯誤的常見原因包括何時停用視頻ID或預定的可用性已過期。

處理bc-catalog-error

正常情況下可能會處理錯誤ready()的部分script阻止可能會導致問題。例如，可能會發生bc-catalog-error事件可以在播放器準備就緒之前分派，如果您在ready()部分，您將無法處理該錯誤。使用地理過濾，未發布視頻，視頻超出計劃範圍或使用其他帳戶時，可能會發生此問題。您可能會發現代碼中沒有問題，但是該問題可能取決於瀏覽器，因此請進行徹底測試。

如果在您的實現中有可能，則您的錯誤處理代碼應使用one()事件處理方法來偵聽bc-catalog-error在單獨的代碼塊中，而不是在ready()部分。

在下面的代碼示例中，您將看到一個代碼塊處理bc-catalog-error（第62行），以及用於ready（第71行）。

使用自定義錯誤

處理特定問題的最佳解決方案bc-目錄錯誤也許使用自定義錯誤。以下代碼顯示了一個插件，該插件使用自定義錯誤通知無效視頻的觀看者為何無法播放該視頻：

videojs.registerPlugin('inactiveErrorCheck', function(options) {
  var myPlayer = this;
  myPlayer.one('bc-catalog-error', function(){
    var specificError;
    myPlayer.errors({
      'errors': {
        'inactive-error': {
          'headline': options.inactiveMessage,
          'dismiss': false
        }
      }
    });
    if (typeof(myPlayer.catalog.error) !== 'undefined') {
      specificError = myPlayer.catalog.error.data[0];
      if (specificError !== 'undefined' & specificError.error_code == "VIDEO_NOT_PLAYABLE") {
        myPlayer.error({code:'inactive-error'});
      };
    };
  });
});
