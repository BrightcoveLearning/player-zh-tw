指導：播放清單API

在本主題中，您將了解Brightcove Player中可用的播放列表API。該API提供了以編程方式控製播放列表的功能。有關實現播放列表視覺表示的信息（更常見的用例），請參見以編程方式實現播放列表文檔。

概述

以下方法，事件和配置選項可作為API的一部分使用：

autoadvance（）方法

這player.playlist.autoadvance( )方法僅用作設置器，並設置播放列表自動前進。啟用後，它將在視頻結束時等待指定的秒數，然後自動繼續播放下一個視頻。任何不是正整數的值都將被視為取消和重置自動前進的請求。如果您在超時時間內更改了自動前進功能，則自動前進將被取消，並且不會前進到下一個視頻，但是它將對以下視頻使用新的超時值。

範圍：
  • 暫停：一個整數（以秒為單位），延遲延遲以將下一個視頻加載到播放列表中
返回值：

該方法不返回值。

該方法的示例用法如下：

player.playlist.autoadvance(0); Will not wait before loading the next item

player.playlist.autoadvance(5); Waits 5 seconds before loading the next item

player.playlist.autoadvance(null); Resets and cancels the auto advance

contains（）方法

這player.playlist.contains( item )方法檢查播放列表中對像是否存在item範圍。如果item在播放列表中true返回（如果不存在）false返回。

參數：
  • 物品：代表播放列表中視頻元素的值。

item參數的有效形式包括：

  • 視頻的字符串形式的URL 
    '//media.w3.org/2010/05/sintel/trailer.mp4'
  • 包含對象的數組，該對象的源和類型已定義 
    [{
  src: '//media.w3.org/2010/05/sintel/poster.png',
  type: 'image/png'
}]
  • 包含來自sources數組的項目的對象 
    {
  sources: [{
    src: '//media.w3.org/2010/05/sintel/trailer.mp4',
    type: 'video/mp4'
  }]
}
  • 在視頻雲中，您可以使用currentSrc()的方法contains()確定當前播放的視頻來自當前加載的播放列表的方法。 
    myPlayer.playlist.contains(myPlayer.currentSrc());
返回值：

該方法返回一個類型的對象布爾值 .

範圍：
  • 無

currentItem（）方法

這player.playlist.currentItem(index)方法既可以用作獲取方法，也可以用作設置方法，它可以根據索引從播放器中播放的播放列表中檢索當前視頻，或者分配新的視頻以在播放器中播放。如果在不帶參數的情況下調用該方法，則該方法將執行檢索並充當獲取程序，而帶參數的參數將分配值並充當設置器。

參數（設置器）：
  • 指數：一個數字，表示要加載到播放器中的視頻的播放列表中的從零開始的數組索引。
返回值（getter）：

該方法返回類型數字的值。返回的值表示當前在播放器中播放的視頻的數組位置。如果播放器當前正在播放非播放列表視頻，currentItem()將返回-1 .

在函數中跳轉到特定視頻的方法的示例用法如下：

  myPlayer.playlist.currentItem(2);
}

playlist（）方法

這player.playlist(newPlayList)方法既可以用作獲取程序，也可以用作設置程序，既可以獲取播放器中當前正在播放的播放列表，也可以將播放列表分配給要播放的播放器。如果在不帶參數的情況下調用該方法，則該方法將執行檢索並充當獲取程序，而帶參數的參數將分配值並充當設置器。

參數（設置器）：
  • 新播放列表：一組視頻源/對象。可以手動創建對象，也可以只是視頻雲播放列表ID或參考ID。

雖然您可以使用playlist()作為設置方法，最佳實踐是利用Brightcove Player的目錄功能加載播放列表，例如 myPlayer.catalog.getPlaylist( playlistID )和myPlayer.catalog.load( playlist )。見玩家目錄文檔以獲取更多信息。

如果您希望加載播放列表，但沒有將播放列表中的第一個視頻放置在播放器中，您可以在該playlist()方法中將第二個參數設置為-1。例如，以下代碼檢索播放列表，將其放入播放器中，但不將第一個視頻加載到播放器中： 
myPlayer.catalog.getPlaylist(playlistID, function(error, myPlaylist) {
  // deal with error
  // +++ Place playlist in player, but -1 parameter does not load playlist video in player +++
  myPlayer.playlist(myPlaylist, -1);
});
當然，然後您必須手動加載視頻。見Brightcove Player示例：在播放清單前播放視訊，以取得此技術的範例。

reverse（）方法

這player.playlist.reverse( )方法會更改播放列表中視頻的順序，以使第一個視頻成為最後一個，而最後一個視頻成為第一個。

  • 無
返回值：

顛倒播放列表中視頻的順序。例如，第一個視頻成為最後一個，最後一個視頻成為第一個。

該方法調度playlistsorted反轉後發生事件。

shuffle（）方法

這player.playlist.shuffle( )方法隨機播放/隨機排列播放列表項的順序。

範圍：
  • 可選的rest選項：默認情況下，整個播放列表是隨機的。但是，在所有情況下（例如用戶已經在觀看視頻時），這可能都不是所希望的。什麼時候true傳遞給此選項，它將僅在當前項目之後隨機播放播放列表項目。例如，在第一個項目上時，該方法將僅對第二個項目及之後的項目進行洗牌。使用該參數的語法是： 
    player.playlist.shuffle({rest: true});
返回值：

該方法不返回值。

此方法使用Fisher-Yates隨機播放算法。

該方法調度playlistsorted改組後的事件。

如果您希望在播放器首次加載時隨機播放播放列表，則必須使用此功能 事件處理程序中的方法playlistsorted事件。這樣可以防止這種情況，即方法嘗試在將播放列表加載到播放器之前先對其進行隨機播放。示例語法為：

myPlayer.on("duringplaylistchange", function() {
  myPlayer.playlist.shuffle();
});

這player.playlist.sort( )方法以與JavaScript相同的方式對播放列表中的視頻進行排序Array.prototype.sort（）方法。

範圍：

beforeplaylistitem事件

這beforeplaylistitem在切換到播放列表中的新內容源之前（即，當currentItem()，first()，等等），但在更改播放器狀態之前。

在playlistchange事件中

這duringplaylistchange調用時更改播放列表的內容後調度事件playlist()，但在更改當前播放列表項之前。事件對象具有幾個特殊屬性：

  • nextIndex：下一個播放列表中將首先播放的索引。
  • nextPlaylist：下一個播放列表的淺表副本。
  • previousIndex：上一個播放列表的索引（此事件觸發時將始終與當前索引匹配，但出於完整性考慮而提供）。
  • previousPlaylist：前一個播放列表的淺表副本。
每次都會觸發此事件player.playlist()被調用，包括第一次。

注意事項

在觸發此事件期間，播放列表被視為處於更改狀態，具有以下效果：

  • 調用主要播放列表方法，player.playlist([...])會拋出一個錯誤。
  • 播放列表導航方法，first，last，next和previous，使其無法使用。
  • 這currentItem()方法僅用作吸氣劑。
  • 在排序方法上，sort，reverse和shuffle，將繼續工作，他們不會派遣playlistsorted事件。

為什麼要使用這個活動

此事件提供了在播放器上設置新來源之前以及播放器之前攔截播放列表設置過程的機會。playlistchange事件觸發，同時提供一致的播放列表API。一個用例可能是改組剛剛來自服務器的播放列表，但是在將其初始源加載到播放器或更新播放列表UI之前：

player.on('duringplaylistchange', function() {
  // Remember, this will not trigger a "playlistsorted" event!
  player.playlist.shuffle();
});

player.on('playlistchange', function() {
  videojs.log('The playlist was shuffled, so the UI can be updated.');
});

播放列表更改事件

這playlistchange每當更改播放列表時，都會異步調度事件。調度此事件後，播放器將開始在新的播放列表中加載第一個視頻。它是異步觸發的，以使瀏覽器開始在新的播放列表中加載第一個視頻。

播放列表項事件

這playlistitem當切換到播放列表中的新內容源時（即，當currentItem()，first()，等等）被調用後，播放器的狀態已更改，但恢復播放之前。

播放列表排序的事件

這playlistsorted當調用任何更改播放列表項順序的方法時調度事件，sort()，reverse()或者shuffle()。

playOnSelect

該值是一個配置選項，當設置為true即使播放器中的當前視頻先前已暫停，也會導致從播放列表中單擊的視頻開始播放。默認情況下，單擊播放列表中的新視頻將加載新視頻，但如果播放器已事先暫停，則將其保持在暫停狀態。

有關操縱此值的信息，請參見播放器配置指南-播放列表部分。
