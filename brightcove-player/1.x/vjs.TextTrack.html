停用 ()

停用：關閉/停用模式 (0) 表示文字軌未處於作用中狀態。除了為了在 DOM 中公開軌道之外，用戶代理忽略了文本軌道。沒有任何線索是活動的，沒有事件被觸發，用戶代理將不會嘗試獲取軌道的線索。

選項 (OBJ)

選項物件的深度合併

每當一個屬性是兩個選項對象的對象，這兩個屬性將使用 vJS.obj.Deepmerge 合併進行合併。

這用於合併子元件的選項。我們希望它很容易覆蓋子組件上的單個選項，而不必重寫所有其他默認選項。

父母原型。選項 _ = { children: { 'childOne': { 'foo': 'bar', 'asdf': 'fdsa' }，'兩個孩子'：{}，'孩子三'：{} }} 新選項 = { children: { 'childOne': { 'foo': 'baz', 'abc': '123' }  '兒童二'：空，'兒童四'：{} }} 這個。選項（新選項）;

結果

{ children: { 'childOne': { 'foo': 'baz', 'asdf': 'fdsa', 'abc': '123' }，'孩子兩個'：空，//禁用。不會被初始化。'三個孩子'：{}，'兒童四'：{} }}
參數：
  • 對象目的新選項值的對象
傳回：
  • 物件合併此 .option_ 和 obj 的新物件

顯示 ()

顯示：模式顯示 (2) 表示文字軌跡處於作用中狀態。如果尚未嘗試獲得軌道的提示，用戶代理將暫時執行這樣的嘗試。用戶代理正在維護一個列表中的線索處於活動狀態，並且事件正在相應地觸發。此外，對於類型為字幕或字幕的文本軌道，線索會適當地顯示在視頻上; 對於類型是描述的文本軌道，用戶代理將提供給用戶以非視覺方式提供線索; 對於類型為章節的文本軌道，用戶代理是為使用者提供機制，使用者可以透過選取提示來導覽至媒體資源中的任何一點。默認情況下顯示狀態與軌道元素上的默認屬性一起使用，以指示文本軌道是由於該屬性而啟用的。這允許用戶代理在根據用戶的喜好設置發現更適合稍後的軌道時覆蓋狀態。
