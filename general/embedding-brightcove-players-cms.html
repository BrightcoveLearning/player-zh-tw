介紹

當您的 CMS 用戶在頁面中嵌入視頻或播放列表播放器時，您希望在生成 Brightcove 播放器嵌入代碼時為他們提供盡可能多的靈活性，以便他們可以控製網頁上 Brightcove 播放器的外觀，但不要指望他們是 HTML 或 JavaScript 專家。

本主題提供了有關如何執行此操作的指南。本文檔的受眾主要是 Brightcove 合作夥伴，但對於將 Video Cloud 或獨立 Brightcove Player 與您的 CMS 集成的任何人都非常有用。

播放器嵌入對話框行為

應該有一個對話框允許用戶控製網頁上播放器的格式。此對話框應允許用戶設置下面列出的字段。假設是在顯示此對話框之前用戶已經選擇了 Brightcove 帳戶和視頻或播放列表。否則，應首先按照以下步驟選擇Brightcove帳戶和視頻或播放列表：附錄A以下。

  1. 播放器：用戶應該能夠從可用的 Brightcove 播放器中選擇播放器。如果他們嵌入了視頻播放器，則只有未啟用播放列表的播放器才可以選擇。如果他們嵌入了播放列表播放器，則只有啟用了播放列表的播放器才可以選擇。
  2. 自動播放：用戶應該能夠啟用/禁用自動播放。默認設置為禁用。
  3. 沉默的：用戶應該能夠啟用/禁用靜音。默認設置為禁用。
  4. 嵌入類型：用戶應該能夠在 iFrame 和 Javascript 嵌入代碼之間進行選擇。
    1. 對於視頻播放器，默認值應為Javascript。
    2. 對於播放列表播放器，默認值應為iFrame。
  5. 漿紗：用戶應該能夠在響應式之間進行選擇 [ 1-1] 或固定尺寸 [ 1-2] .
    1. 對於視頻播放器，默認值應為自適應。
    2. 對於播放列表播放器：
      1. 如果是iFrame，則默認值應為自適應。
      2. 如果使用Javascript，則默認值為固定。
  6. 長寬比：用戶應該能夠選擇縱橫比 [ 1-1] .
    1. 選擇是：
      • 16:9
      • 4:3
      • Custom
    2. 默認值應為16:9。
  7. 寬度、高度 :用戶應該能夠輸入寬度和高度 [ 1-1] .
    1. 如果寬高比為16：9或者4：3， 這高度該字段應顯示為灰色，並根據以下內容自動計算寬度和長寬比。
    2. 默認值應為640為了寬度和360為了高度當“寬高比"為“自定義"時。

筆記

  • [ 1-1] 如果 CMS 提供用於嵌入代碼的本機容器，允許用戶指定響應式與固定式和寬度/高度，則使用容器來調整大小並讓 Brightcove 嵌入代碼始終具有響應性可能更有意義。
  • [ 1-2] 下麵包含的響應式嵌入代碼使用寬度作為最大寬度。

嵌入參數和代碼

以下部分描述瞭如何根據用戶選擇生成 Brightcove 播放器嵌入代碼。這%XYZ%值被替換到嵌入代碼中，如下所述。

常用參數

  • %AUTOPLAY% = autoplay if Autoplay enabled, else it is a null string
  • %MUTED% = muted if Muted enabled, else it is a null string
  • %ACCOUNTID% = User selected Brightcove account ID
  • %VIDEOID% = User selected video ID
  • %PLAYLISTID% = User selected playlist ID
  • %PLAYERID% = User selected Video player ID or Playlist player ID
  • %CMS% = Name of CMS
  • %CMSVERSION% = Version of CMS
  • %CONNECTORVERSION% = Version of connector

嵌入iFrame視頻播放器

自適應大小調整的參數（最大大小）

%MAXWIDTH% = Width . ‘px’
%MINWIDTH% = 0px
%WIDTH% = 100% %HEIGHT% = 100%
%PADDINGTOP%
  if (Aspect Ratio == “16:9”) {
    %PADDINGTOP% = 56%
  } elseif (Aspect Ratio == “4:3”) {
    %PADDINGTOP% = 75%
  } elseif {
  %PADDINGTOP% = (Height / Width * 100) . “%”
  }

固定尺寸的參數

%MINWIDTH% = Width . ‘px’
%WIDTH% = Width . ‘px’
%HEIGHT% = Height . ‘px’
%PADDINGTOP%
  if (Aspect Ratio == “16:9”) {
    %PADDINGTOP% = 56%
  } elseif (Aspect Ratio == “4:3”) {
    %PADDINGTOP% = 75%
  } elseif {
  %PADDINGTOP% = (Height / Width * 100) . “%”
  }

Brightcove嵌入代碼

<div style="display: block; position: relative; min-width: %MINWIDTH%; max-width: %MAXWIDTH%;">
  <div style="padding-top: %PADDINGTOP%;">
    <iframe src="//players.brightcove.net/%ACCOUNTID%/%PLAYERID%_default/index.html?videoId=%VIDEOID%
      &usage=cms:%CMS%:%CMSVERSION%:%CONNECTORVERSION%:iframe
      &%AUTOPLAY%
      &%MUTED%"
      allowfullscreen=””
      webkitallowfullscreen=””
      mozallowfullscreen=””
      style= width: %WIDTH%; height: %HEIGHT%; position: absolute; top: 0; bottom: 0; right: 0; left: 0;">
    </iframe>
  </div>
</div

附錄A

如果在打開播放器嵌入對話框之前未選擇視頻或播放列表，則應該在對話框中選擇視頻或播放列表（如有必要，還有Brightcove帳戶）。該對話框應向用戶顯示下面列出的字段：

  1. Brightcove帳戶：如果尚未選擇 Brightcove 帳戶，則用戶應該能夠選擇該帳戶。
  2. 搜索過濾器：用戶應該能夠輸入搜索字符串來過濾顯示的視頻或播放列表。如果使用Brightcove搜索API，則搜索字符串應為URI編碼（請參見下面的示例）。
  3. 文件夾：用戶應該能夠從 Brightcove 帳戶中選擇一個文件夾名稱來過濾顯示的視頻列表。Brightcove API 沒有為文件夾提供內置過濾器，因此客戶端代碼需要檢索所有視頻，然後按文件夾過濾。（不適用於播放列表）
  4. 限制：用戶應該能夠限制返回的視頻數量，主要是為了提高搜索性能。使用 Brightcove 搜索時，可以指定的最大限制為 100。要返回超過 100 個視頻，必須實現分頁機制。另外，如果用戶選擇了要過濾的文件夾，則客戶端需要使用分頁機制請求帳戶中的所有視頻，然後按文件夾在本地過濾該列表，並返回用戶限制指定的視頻數。(Does not apply to playlists)
  5. 排序方式：用戶應該能夠選擇排序字段。
    1. 對於視頻，用戶應該能夠選擇視頻名稱，更新日期，創建日期，開始日期和總播放次數。默認應該是更新日期。
    2. 對於播放列表，用戶可以選擇名稱和修改日期。默認應該是修改日期。
  6. 排序：用戶應該能夠選擇升序或降序排序。默認值應該是降序的。

該對話框應具有以下行為：

  1. 根據上面的用戶選擇顯示視頻或播放列表的列表。允許用戶選擇一個視頻或一個播放列表。
  2. 顯示視頻列表時：
    1. 只應列出活動視頻。
    2. 應顯示縮略圖、視頻名稱和 ID。
    3. 只能選擇一個視頻。
  3. 顯示播放列表時：
    1. 如果播放列表是手動播放列表，請顯示播放列表名稱，ID和播放列表中的視頻數。
    2. 如果播放列表是智能播放列表，請顯示播放列表名稱，ID和-代替視頻數量。
    3. 只能選擇一個播放列表。
  4. 顯示可點擊的鏈接，該鏈接將在新的瀏覽器標籤中打開視頻或播放列表播放器。

URI 編碼示例

  1. 搜索字符串：great+blue

    串入內容管理系統 API調用應該是：great%2Bblue

    返回的視頻：所有包含以下詞的視頻：great AND blue[2-1]

  2. 搜索字符串：great blue

    串入內容管理系統 API調用應該是：great%20blue

    返回的視頻都是包含以下詞的：great OR blue[2-1]

  3. 搜索字符串："great blue"

    串入內容管理系統 API調用應該是：%22great%20blue%22

    返回的視頻都是包含完整短語的：great blue[2-1]
