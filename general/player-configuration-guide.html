介紹

您可以在播放器配置中設置的屬性詳述如下（並在左側導航中列出）以帳戶ID .本文檔中還包括有關如何查看和配置屬性值的信息。

你不應該使用data-setup與 Brightcove 播放器。你可能會看到使用data-setup在 API 文檔中，但這是因為該文檔是直接從 Video.js 播放器源代碼生成的，您可以將該屬性與video使用純 Video.js 標記。該屬性向播放器發送配置信息，但 Brightcove Player 使用不同的方法來執行此任務，這使得data-setup不可靠。

瀏覽使用config.json

新的Brightcove播放器配置的實例存儲在JSON文件中。您可以通過多種方式查看此 JSON 文件或其中的一部分。

查看播放器配置的一種方法是通過將播放器的URL更改為index.html至config.json。例如：

  http://players.brightcove.net/1507807800001/HJyMlHiuZ_default/config.json

播放器配置中的一些屬性定義了以下內容：

  • 播放器的名稱和版本
  • 播放器將播放的視頻
  • 玩家所屬的賬號
  • 玩家身份證
  • 播放器可以使用的任何插件，以及插件代碼位置的網址
  • 影片播放前要展示的海報

帳戶ID

與播放器關聯的 Video Cloud 帳戶 ID。這個值無法修改 .

ad_config_id

ad_config_id information:

  • 數據類型：字符串
  • 默認：undefined
  • 動態投放SSAI廣告配置ID。除了在播放器配置上進行設置外，還可以在運行時通過iframe播放器的adConfigId查詢字符串參數或頁內播放器的data-ad-config-id屬性進行設置。這些都將覆蓋播放器配置中的任何值。

設置值的其他方式：

名稱 類型 嵌入類型 行為
數據廣告配置ID HTML屬性 頁內 覆蓋配置
廣告配置 ID 查詢參數 iframe 覆蓋配置

ad_failover

ad_failover information:

  • 數據類型：布爾值
  • 默認：undefined
  • 設置為 true 以啟用廣告故障轉移行為。這是一個多部分配置，取決於：
    • 必須在播放器上配置IMA3和SSAI插件。
    • 一個ad_config_id應該可用，否則SSAI廣告不可用。

設置為真的，播放器將延遲其自動初始化過程，直到廣告塊檢測已做出決定為止。這可能會延遲播放器初始化最多 100 毫秒左右。

application_id

application_id information:

  • 數據類型：字符串
  • 默認：undefined
  • 應用程序 ID 用於在指標中區分單個播放器的多次使用。用戶看不到它。

設置值的其他方式：

名稱 類型 嵌入類型 行為
數據應用程序ID HTML屬性 頁內 覆蓋配置
應用程序ID 查詢參數 iframe 覆蓋配置

自動前進

autoadvance information:

autoplay information:

base_url細節

在北美以外使用Brightcove播放器使用IP限制時，可能會出現問題。當 Brightcove 播放器被要求從 Video Cloud 帳戶加載視頻時，它會連接到此地址的目錄https://edge.api.brightcove.com/playback/v1/ .

該 Edge 端點的行為方式可能會導致 IP 限制問題：

  • 從WITHIN North America打Edge時，直接打Edge，Edge看到的是用戶的IP地址。它可以根據IP限制白名單允許或拒絕對目錄的訪問。這是所需的功能。
  • 當從 OUTSIDE North America 訪問 Edge 時，出於負載平衡/本地緩存 CDN 的原因，它會通過 Cloudfront。這很重要，因為因此，Edge 看到的是 Cloudfront IP 而不是用戶的 IP，並拒絕所有人，因為 Cloudfront 不在白名單上（而且永遠不應該在白名單上，因為那樣任何使用 Cloudfront 的人都可以訪問）。

要解決此問題，您可以將播放器配置為使用備用 Edge 端點， edge-elb.api.brightcove.com .必須對需要從北美以外訪問 IP 受限帳戶的每個玩家進行此更改。

這種方法有一個缺點。從美國以外地區檢索緩存內容的時間將稍有延遲，因為目錄請求將發送到弗吉尼亞州，而不是通過Cloudfront的本地服務器全球網絡發送。這很可能幾乎不會引起注意。

實際的配置更改涉及為 Video Cloud 設置一個值base_url .請記住，僅當北美以外的Brightcove播放器使用IP限制時，才應更改此設置。對於現有的播放器，您可以使用以下curl命令，該命令使用PATCH HTTP方法（您必須將數據替換為環境變量$EMAIL，$ACCOUNT_ID和$PLAYER_ID）：

curl
  --header "Content-Type: application/json"
  --user $EMAIL
  --request PATCH
  --data '{
      "video_cloud": {
        "base_url": "https://edge-elb.api.brightcove.com/playback/v1/"
      }
    }'
  https://players.api.brightcove.com/v2/accounts/$ACCOUNT_ID/players/$PLAYER_ID/configuration

"prefer_mp4": true
使用時，所有非 MP4 媒體源將在傳遞給播放器之前從源數組中過濾掉，就好像它們從未從播放 API 返回一樣。

發布者還可以通過提供一個數組來控制此過濾源列表的排序。這僅支持 Playback API 返回的數字屬性（例如大小、高度、寬度等）。在這種情況下，數組的第一個元素應該是排序的屬性，第二個可選元素應該是排序的方向，或者“上升”或者“降級” .默認值為“降級” .例如，要過濾掉非 MP4 媒體源並按大小從最小到最大排序，您可以使用：

"video_cloud": {
  "source_selection_rules": {
    "prefer_mp4": ["size", "asc"]
  }
}

體積方向

volume_orientation information:

  • 數據類型：字符串
  • 默認值：水平
  • 僅適用於 Brightcove Player 6.32.0 及更新版本
  • 這volume_orientation屬性確定音量滑塊是水平還是垂直方向。

有效值為：

價值 描述
"vertical" 顯示垂直音量面板（這相當於傳遞選項對象controlBar.volumePanel.inline=false和controlBar.volumePanel.vertical=true到公元前（）方法或數據設置屬性）。
"horizontal" 顯示水平音量面板（luna 和石墨皮膚中的默認音量面板方向）
未定義 將此保留為未定義將顯示水平音量面板（luna 和石墨皮膚中的默認音量面板方向）

您可以使用 Studio 或 curl 來更改屬性的值：

    --user $EMAIL \

        "volume_orientation": "vertical"
       }' \

虛擬機

vttjs information:

  • 數據類型：字符串
  • 默認值：vjs.zencdn.net CDN
  • 這與標準 Video.js 選項相同vtt.js .Brightcove 播放器使用虛擬機相反，因為在播放器管理 API 中無法使用帶點的屬性名稱，並且在其他地方可能會出現問題。
  • 可以設置為數據-vttjs屬性，一個虛擬機 iframe 查詢參數，一個虛擬機設置選項或虛擬機在播放器配置中。

wait_for_ad_block_detect

wait_for_ad_block_detect information:

  • 數據類型：布爾值
  • 默認值：未定義
  • 當設置為true，播放器將延遲其自動初始化過程，直到廣告塊檢測做出決定之後。這可能會延遲播放器初始化最多 100 毫秒左右。這不需要設置為true使用時ad_failover因為在這兩種情況下都會發生延遲行為。
