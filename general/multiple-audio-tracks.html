多個音軌

{{ page.description }}

介紹

Brightcove 播放器支持具有多個備用音軌的視頻。如果視頻準備了多個備用音軌，則會出現一個特殊的耳機圖標 () 自動出現在控制欄中。同樣，如果您具有正確編碼的視頻，則將出現該圖標，您無需執行其他任何操作。單擊圖標後，視頻的觀看者可以在音軌之間進行選擇。

筆記：
  • 僅啟用了動態交付的帳戶才支持多個音頻軌道，並且必須專門為多個音頻軌道啟用該帳戶。有關詳細信息，請聯繫您的客戶經理。
  • 僅VOD內容支持多個音軌。
  • 為一個帳戶啟用多個音軌後，任何具有多個音軌的現有視頻源將需要重新轉碼。
  • 當您向視頻中添加其他音軌時，它們必須都具有相同的比特率/再現配置文件，以確保在HLS中成功播放（DASH播放仍將起作用）。
  • 平滑目前不支持多個音軌。
  • 上載具有多個音軌的源視頻文件將在轉碼後僅生成一個音軌。通常，將其中一個軌道標記為默認軌道，但是如果沒有默認軌道，則Ingest系統會根據立體聲質量和最高比特率選擇“最佳"音頻軌道，並將其標記為en / main。

例子

單擊播放後，您將在控制欄中看到耳機圖標，從中可以在兩個替代音軌之間進行選擇。

用例

以下是多個替代音軌的一些常見用例：

  • 多種語言支持
  • 立體聲與多聲道（環繞聲）音頻
  • 評論曲目
  • 描述性音頻

樣式圖標和菜單

如果默認顏色不能滿足您的需要，則可以更改耳機圖標，所選選項和其他選項的顏色。下表和屏幕截圖詳細說明了您將需要使用的選擇器。（請注意，屏幕截圖中顯示的顏色不是默認值，而是易於識別的顏色。）

音頻圖標選擇器
圖元 選擇器
耳機圖標（屏幕截圖中的紅色） .video-js .vjs-control-bar .vjs-audio-button
選定的選項（屏幕截圖中為黃色） .video-js .vjs-control-bar .vjs-audio-button .vjs-menu-item.vjs-selected
可用選項（屏幕截圖中的綠色） .video-js .vjs-control-bar .vjs-audio-button .vjs-menu-item

以下為實際CSS影響屏幕截圖中的更改：

.video-js .vjs-control-bar .vjs-audio-button {
  color: red;
}
.video-js .vjs-control-bar .vjs-audio-button .vjs-menu-item.vjs-selected {
  background-color: yellow;
}
.video-js .vjs-control-bar .vjs-audio-button .vjs-menu-item {
  background-color: green;
}

已知的問題

  • 如果您通過<source>標籤在播放器中，觀看者將無法在Android OS的Chrome中選擇多個音軌。
