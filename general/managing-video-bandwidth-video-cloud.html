帶寬使用基礎

像任何其他數字資產一樣，視頻也作為數據通過Internet傳輸到瀏覽器和應用程序。傳輸的數據量是消耗的帶寬，需要付費。此帶寬可以分為兩部分：

  1. 使用的帶寬：與觀看者觀看的視頻部分相關的數據
  2. 開銷帶寬：與已下載但未實際觀看的視頻部分相關的數據-始終會有一些開銷，因為：
    • 視頻數據是 緩衝的 在觀看進度之前，以便在下載其他數據時視頻不會繼續暫停
    • 在某些情況下設備 預載 部分或全部視頻數據（取決於視頻文件格式）在觀眾嘗試播放之前 - 原因是設備可用的網絡帶寬通常小於計算機可用的帶寬，因此預加載允許視頻當觀眾點擊播放按鈕時開始播放更快

增加帶寬

帶寬使用率正在上升！發生這種情況的原因有幾個：

  1. 更多觀眾：總體而言，通過計算機和/或設備連接到Internet的人數正在增加，並且他們在線觀看視頻的數量也在增加。當然，這是全球性的增長，並且在特定站點和應用程序上會有所不同。您可以判斷是否有更多的觀眾使用Video Cloud Analytics查看諸如視頻展示次數，視頻觀看次數和唯一身份用戶之類的指標。
  2. 更多互聯網帶寬：由於支持互聯網的基礎架構和網絡協議的改進，平均而言，互聯網用戶擁有更多可用帶寬（同樣，這是全球平均值，並且存在局部差異）。這對於在線視頻有兩個含義：
    • 無論用戶是否觀看視頻，都可以更快地下載更多視頻數據
    • 更快的下載時間意味著在很多情況下都可以查看高質量的演繹版
  3. 互聯網的使用正在向設備轉移：這包括在線視頻的消費。由於上述預加載行為，在某些情況下，這往往會增加帶寬消耗。此外，屏幕質量，內存和設備網絡功能的改進意味著觀看者可以觀看比過去更高質量的視頻。
  4. 視頻雲的變化：這裡最重要的變化是引入了攝取個人資料，為您為不同類型的視頻創建的演繹集提供更大的靈活性。在標準攝取檔案響應於上述更改，隨著時間的推移，視頻播放次數已增加，這使許多用戶可以使用更高質量的視頻。

如果這些因素增加 已用帶寬， 這是一件好事，因為這可能意味著您有更多的觀眾觀看您的視頻的更高質量的再現。這些增加不可避免地也意味著開銷帶寬的增加，而且可能是開銷增加的幅度足夠大，您可能希望減少開銷。後面的主題各節討論影響帶寬消耗的各種因素，以及可以選擇減少帶寬的選項。

您不一定需要做任何事情。Brightcove配置Brightcove播放器和攝取配置文件，以向觀看者提供最高質量的視頻體驗。此處描述的選項將在某種程度上影響體驗，以減少所消耗的視頻帶寬。如果您希望最小化視頻帶寬，您可以嘗試使用下面描述的選項，看看在保持可接受的觀看體驗質量的同時是否可行。

文件類型

在後Flash世界中，視頻文件是通過HTTP連接傳遞到瀏覽器和設備的，文件類型決定了該操作的完成方式。MP4再現作為完整文件提供，而按時間分段的再現（HLS或DASH）則根據需要以短段形式提供。這種差異可能會對開銷帶寬產生重大影響，尤其是對預加載視頻的設備。如果查看器在頁面或應用程序視圖上停留的時間足夠長，則即使未觀看任何MP4副本，也將下載整個MP4副本。另一方面，HLS再現通常分為10秒片段，因此在同一情況下，將僅下載視頻數據的一部分。

為無法播放分段視頻格式的舊平台提供 MP4 格式仍然很重要，但您可以 顯著減少開銷帶寬 通過確保您的所有視頻都具有一組良好的 HLS 再現（或 DASH，如果您需要）數字版權管理不支持 HLS 的類型）。

預裝

如您所見，默認情況下，如果沒有按時間段細分的版本，大多數設備默認都會立即預加載第一個HLS（或DASH）段-或整個MP4版本。改善了 明顯的 視頻的性能，因為當觀眾點擊播放按鈕時，播放將立即開始。更好的性能通常會提高視頻的播放速度。這樣做的代價是，即使很少或從未播放過的視頻也會佔用視頻帶寬。

是否應該為某些或所有播放器關閉預加載取決於幾個因素：

  • 如果視頻是頁面的焦點，或者您有充分的理由相信用戶會觀看您的視頻，則應保持預加載狀態。
  • 查看您的視頻分析-播放速率極低的視頻是浪費帶寬的最嚴重違法者，因此請考慮關閉提供這些視頻的播放器的預加載。
  • 如果您在同一頁面上有多個播放器，建議您關閉預加載功能，因為觀看所有視頻的可能性較小。

渲染比特率

更高的比特率視頻再現可提供更高質量的視頻，但以消耗更多帶寬為代價。如果在所有情況下都能提供最高質量的視頻是您的頭等大事，那麼這是您必須承受的成本。但是，值得考慮的是，是否可以通過重新轉換視頻並消除某些比特率最高的再現來節省此區域的開銷帶寬（例如，如果您對所有視頻使用高分辨率配置文件，則您正在創建一些非常高的比特率演繹）。

您的視頻分析可以在這裡提供幫助。查看正在觀看視頻的設備類型和設備操作系統。例如，如果視圖主要在電話上顯示，則在wifi連接上的智能電話可能具有足夠的Internet帶寬以使用非常高質量的格式，但是在電話屏幕上，這些外觀可能看起來並不比低質量的格式好得多。最好的方法是從Studio中實際下載視頻的各種格式，並在目標平台上查看它們，以查看最高比特率格式和較低比特率格式之間的質量差異。

正在緩衝

Brightcove播放器會在播放期間自動緩衝視頻-這意味著將下載視頻的其他將來片段，以防止在達到片段末尾時下載其他數據時暫停視頻。默認情況下，播放器嘗試將緩衝視頻數據保持在當前播放位置之前。

對於短視頻，由於緩衝而導致的帶寬開銷成比例地更大。例如，考慮一分鐘的視頻，平均觀看三十秒。那麼，對於每個視圖，所消耗的帶寬就是所使用的帶寬。

緩衝的數量基於播放器的當前時間。該實現的邏輯是，用戶觀看內容的時間越長，他們繼續觀看並完成內容的可能性就越大，因此可以緩衝更多的內容而不會浪費帶寬。邏輯非常簡單，播放器在最小和最大之間線性插值當前時間。默認的最小值和最大值分別為30s和60s。因此，一旦播放器觀看了30秒的內容，就會有大約60秒的前移緩衝區。該實現不跟踪觀看者是否在視頻中向前滑動。

本節僅供參考。Brightcove 內部測試表明，在某些情況下，減小緩衝區大小會顯著降低播放體驗，因此 Player API 沒有公開執行此操作的方法。
