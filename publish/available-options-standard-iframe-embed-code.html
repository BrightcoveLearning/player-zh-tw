    標準 (iframe) 內嵌程式碼的可用選項

    在本主題中，您將了解有關iframe嵌入代碼。此版本的播放器實現也稱為標準播放器代碼。

    簡介

    的iframe播放器實現代碼使播放器易於使用，成為一個單獨的包含單元。這對於社交媒體應用程序和共享通常是必需的。

    用法

    注意：如果您是Brightcove Player客戶，則不會使用videoId查詢參數以將視頻分配給播放器。您可以在播放器模塊中分配視頻或使用 curl。

    要使用iframe代碼遵循以下步驟：

    1. 使用  PLAYER 模組來建立玩家。
    2. 轉到 MEDI A 模塊並使用新創建的播放器發布視頻。
    3. 複製標準 嵌入代碼。HTML 將類似於以下內容： 
            <iframe src="https://players.brightcove.net/921483702001/default_default/index.html?videoId=4570583963001"
        allowfullscreen
        webkitallowfullscreen
        mozallowfullscreen></iframe>
    4. 從瀏覽器複製 HTML，然後貼到完整body的 HTML 頁面。
    5. 瀏覽 HTML 頁面以查看播放器正常運作。

    查詢參數

    許多查詢參數可用於iframe標籤以提供有關玩家應如何表現的其他信息。下表詳細列出了可用參數。

    使用只能為true或false的查詢參數時（數據類型布爾型），您只需添加查詢參數即可將其打開，或排除該參數將其關閉。這裡顯示一個示例，使用muted。

    <iframe src="https://players...index.html?videoId=4784463159001&muted"

    參數 描述 資料類型
    adConfigId 動態傳遞 SSAI 廣告組態識別碼。 字串
    應用程式 ID 允許重複使用單一播放程式，但會區分每個站台或每個應用程式的分析。如需完整詳細資訊，請參閱新增應用程式 ID 至 Player 內嵌程式碼文件。 字串
    自動播放 表示播放器應該立即開始播放，在允許此操作的平台上。

    如果設定autoplay為 Boolean 值，則會使用瀏覽器的原生自動播放行為。如果設定為下列三個有效字串值之一，就會發生下列情況：
    • play：玩家將手動嘗試調用播放。
    • muted：玩家將手動將播放器靜音並呼叫播放。
    • any：玩家會先嘗試呼叫播放。如果失敗，請將播放器靜音並呼叫播放。
    如果上述任何一項失敗，玩家將顯示「大遊戲按鈕」，就好像自動播放設定為 false 一樣。

    讓視頻自動播放可能是一個複雜的情況。如需詳細資訊，請參閱「自動播放考量」文件。
    如果自動播放在播放器級別啟用（使用Studio或播放器管理API），無法禁用它。例如，&autoplay=false不會停止自動播放。實際上，最後一個查詢參數實際上會打開自動播放，就好像URL包含字符串一樣自動播放，則無論啟用了什麼值，都將啟用自動播放功能。
    		 布林值或字串
    crossorigin 如果為 true ，底層 < 視頻 > 元素將具有交叉源 =「匿名」。這意味著任何拉入播放器的視頻或文本軌道都必須有 CORS 標題。
    注意：只有當您知道您的所有內容（視頻和文本軌道）都應用了 CORS 標題時，才應該使用此功能。
    		 布林值
    language 設定播放程式的語言 (使用有效的語言代碼，通常是兩個字母)。 字串
    loop 使視頻在結束後立即重新開始。 字串
    muted 將視訊的聲音靜音。 布林值
    playlistId 設定播放器中播放清單的 ID 或參考 ID。 字串
    playlistVideoId 設定要在視訊中播放的初始視訊，此視訊必須位於指定的播放清單中。 字串
    playsinline 如果平台和操作系統允許，它將在播放器的播放區域內顯示視頻內容，這意味著它不會全屏顯示或在可調整大小的獨立窗口中顯示；可以在iPhone和iPad上使用 布林值
    預先載入 通知瀏覽器視頻數據是否應該在加載視頻標籤後立即開始下載。可能的值為none、metadata或auto。如需進一步的詳細資訊，請參閱下一節。 字串
    推薦人 允許您傳遞document.referrer以及iframe的其他參數，可讓您準確跟踪視頻的引薦來源網址。請注意，您將需要對值進行URI編碼-例如：referrer=http%3A%2F%2Ffoo.com。另請注意，此參數可用於播放器的6.25版本或更高版本。 字串
    t 設定視訊開始播放的時間偏移量。如需詳細資訊，請參閱深層連結文件。 字串
    videoId 設定播放器中視訊的 ID 或參照 ID。 字串

    預先載入詳細資訊

    preload參數通知瀏覽器，在加載視頻標籤後，是否應立即開始下載視頻數據。選項為自動、中繼資料和無選項。對於 HLS 播放技術，設置意味著稍微不同的事情，並在下面說明。

    • 汽車（默認）：立即開始載入視訊 (如果瀏覽器同意)。某些行動裝置 (例如 iPhone 和 iPad) 不會預先載入影片，以保護使用者的頻寬。這就是為什麼這個值被稱為 auto ，而不是像真實的更終的東西。

      對於 HLS 播放技術，自動表示播放程式下載 m3u8 資訊清單檔案，而只有 1 個區段而非 3-4，就像在 5.3 版本之前的 Brightcove 播放程式一樣。

    • 元數據：只載入視訊的中繼資料，其中包括視訊的持續時間和尺寸等資訊。

      對於 HLS 播放技術，中繼資料與自動相同。

    • 無：請勿預先載入任何視訊資料。這將等到用戶點擊播放開始下載。請注意，在 iOS 上，即使使用此設定，也可能會下載區段。如需詳細資訊，請參閱布萊特灣播放器和 iOS 文件。

      對於HLS播放技術，沒有意味著玩家將不再加載任何東西，包括m3u8清單，直到用戶按下play為止。
          <video-js preload ...>
      or
      { "preload": "auto" }
    注意：使用伺服器端廣告插入 (SSAI) 時，Brightcove 建議設定preload=none。如果設定preload為auto或metadata，系統會要求視訊串流的第一個影格，即使使用者未開始播放，也會將廣告計算為已傳送的廣告。
