標準 (iframe) 內嵌程式碼的可用選項

{{ page.description }}

簡介

iframe 播放器實現代碼提供了播放器作為一個包含單個單元的易用性。這通常是社交媒體應用程序和共享必需的。

用法

注意：如果您是 Brightcove Player 客戶，則不會使用videoId查詢參數將視訊指派給您的播放器。您可以在播放器模塊中分配視頻或使用 curl。

若要使用 iframe 程式碼，請依照下列步驟執行：

  1. 使用  PLAYER 模組來建立玩家。
  2. 轉到 MEDIA 模塊並使用新創建的播放器發布視頻。
  3. 複製「標準」 內嵌程式碼。HTML 將類似於以下內容： 
          <iframe src="https://players.brightcove.net/921483702001/default_default/index.html?videoId=4570583963001"
        allowfullscreen
        webkitallowfullscreen
        mozallowfullscreen></iframe>
  4. 從瀏覽器複製 HTML，然後貼到完整body的 HTML 頁面。
  5. 瀏覽 HTML 頁面以查看播放器正常運作。

查詢參數

iframe 標籤提供許多查詢參數，以提供有關玩家應該如何行為的其他資訊。下表詳細說明了可用的參數。

使用只能為 true 或 false (資料類型布林值 ) 的查詢參數時，您只需包含查詢參數即可將其開啟，或排除它以將其關閉。一個例子顯示在這裡使用muted .

<iframe src="https://players...index.html?videoId=4784463159001&muted"

參數 描述 資料類型
管理設定 動態傳遞 SSAI 廣告組態識別碼。 字串
應用程式 ID 允許重複使用單一播放程式，但會區分每個站台或每個應用程式的分析。如需完整詳細資訊，請參閱新增應用程式 ID 至 Player 內嵌程式碼文件。 字串
自動播放 表示播放器應該立即開始播放，在允許此操作的平台上。

如果設定autoplay為 Boolean 值，則會使用瀏覽器的原生自動播放行為。如果設定為下列三個有效字串值之一，就會發生下列情況：
  • play：玩家將手動嘗試調用播放。
  • muted：玩家將手動將播放器靜音並呼叫播放。
  • any：玩家會先嘗試呼叫播放。如果失敗，請將播放器靜音並呼叫播放。
如果上述任何一項失敗，玩家將顯示「大遊戲按鈕」，就好像自動播放設定為 false 一樣。

讓視頻自動播放可能是一個複雜的情況。如需詳細資訊，請參閱「自動播放考量」文件。
如果在播放程式層級啟用自動播放功能 (使用 Studio 或播放程式管理 API)，則無法停用它。例如，不&autoplay=false會停止自動播放。事實上，最後一個查詢參數實際上會打開自動播放，就好像 URL 包含字符串自動播放一樣，自動播放將被啟用，無論可能分配給它的值。
 布林值或字串
crossorigin 如果為 true ，底層 < 視頻 > 元素將具有交叉源 =「匿名」。這意味著任何拉入播放器的視頻或文本軌道都必須有 CORS 標題。
注意：只有當您知道您的所有內容（視頻和文本軌道）都應用了 CORS 標題時，才應該使用此功能。
 布林
language 設定播放程式的語言 (使用有效的語言代碼，通常是兩個字母)。 字串
loop 使視頻在結束後立即重新開始。 字串
muted 將視訊的聲音靜音。 布林
playlistId 設定播放器中播放清單的 ID 或參考 ID。 字串
playlistVideoId 設定要在視訊中播放的初始視訊，此視訊必須位於指定的播放清單中。 字串
playsinline 如果平台和操作系統允許，將在播放器的播放區域內顯示視頻內容，這意味著它不會顯示全屏或獨立的，可調整大小的窗口; 在 iPhone 和 iPad 上是否適用 布林
預先載入 通知瀏覽器視頻數據是否應該在加載視頻標籤後立即開始下載。可能的值為none、metadata或auto。如需進一步的詳細資訊，請參閱下一節。 字串
反向連結 允許您將與其他參數一document.referrer起傳遞給 iframe，以便您準確地追蹤視訊的反向連結 URL。請注意，您需要對值進行 URI 編碼-例如：referrer=http%3A%2F%2Ffoo.com。另請注意，此參數適用於 6.25 版本的播放程式或更新版本。 字串
t 設定視訊開始播放的時間偏移量。如需詳細資訊，請參閱深層連結文件。 字串
videoId 設定播放器中視訊的 ID 或參照 ID。 字串

預先載入詳細資訊

預加載參數通知瀏覽器視頻數據是否應該在加載視頻標籤後立即開始下載。選項是汽車 , 元數據， 和沒有任何 .對於 HLS 播放技術，設置意味著稍微不同的事情，並在下面說明。

  • 自動 (預設值)：立即開始載入視訊 (如果瀏覽器同意)。部分行動裝置 (例如 iPhone 和 iPad) 不會預先載入視訊，以保護使用者' 頻寬。這就是為什麼這個值被稱為 auto，而不是像真實的更終的東西。

    對於 HLS 播放技術，自動表示播放程式下載 m3u8 資訊清單檔案，而只有 1 個區段而非 3-4，就像在 5.3 版本之前的 Brightcove 播放程式一樣。

  • 中繼資料：只載入視訊的中繼資料，其中包括視訊的持續時間和尺寸等資訊。

    對於 HLS 播放技術，中繼資料與自動相同。

  • 無：請勿預先載入任何視訊資料。這將等到用戶點擊播放開始下載。請注意，在 iOS 上，即使使用此設定，也可能會下載區段。如需詳細資訊，請參閱布萊特灣播放器和 iOS 文件。

    對於 HLS 播放技術，無意味著播放器將不再載入任何內容，包括 m3u8 清單，直到用戶按下播放。
      <video-js preload ...>
      or
      { "preload": "auto" }
注意：使用伺服器端廣告插入 (SSAI) 時，Brightcove 建議設定preload=none。如果設定preload為auto或metadata，則視訊的第一個影格串流，即使使用者沒有開始播放，也會將廣告計算為已傳送的廣告。
