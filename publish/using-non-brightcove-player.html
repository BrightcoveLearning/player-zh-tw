使用非Brightcove播放器

{{ page.description }}

介紹

Brightcove Web和移動設備播放器具有直接或通過插件內置的大量功能，可以處理您的Video Cloud帳戶的各個方面：

  • 檢索視頻數據和元數據
  • 通過域或地理限制處理內容保護
  • 將分析數據發送到Brightcove數據收集器

如果出於某些原因需要播放視頻雲視頻，則需要自己處理此功能。以下各節說明瞭如何使用Brightcove API來執行此操作。

獲取策略密鑰

為了獲取視頻和播放列表數據或設置內容限制，您需要政策金鑰。在大多數情況下，您可以通過Video Cloud Studio進行此操作，也可以使用策略API。

簡單的策略密鑰

如果您需要做的就是按ID檢索視頻和/或播放列表，則只需從玩家那裡獲得策略密鑰。

如果你想能夠搜索視頻使用播放API，您將需要創建一個啟用搜索的策略密鑰。您可以通過Policy API進行操作，也可以使用我們嵌入在快速開始：政策API。

具有播放限制的策略密鑰

播放器的域限制和地理位置限制也使用策略密鑰來實現。如果您需要一個或兩個，則獲取策略密鑰的最簡單方法是在Studio中對新播放器設置所需的限制，然後從該玩家那裡獲得策略密鑰。如果您願意，也可以使用以下命令直接生成此策略密鑰：政策API。

獲取視頻數據

有了策略密鑰後，您可以使用播放API。可以通過三種方式檢索數據：

  • 通過ID或reference_id獲取視頻
  • 通過ID或reference_id獲取播放列表
  • 取得影片搜索

見快速開始：播放API對於示例JavaScript，您可以用來設置API請求。

響應數據將包括視頻sources，它是一個演繹數據數組，包括URL。您還將獲得縮略圖和海報圖像的URL，以及視頻等元數據name和description。

如何將這些數據加載到播放器中將取決於播放器的API。

分析

由於非Brightcove播放器不會自動將數據發送到資料收集API對於Brightcove Analytics系統，實施該程序是您的責任。

您需要執行的操作取決於播放器的API，尤其是播放器動作廣播的事件。見數據收集API樣本基於VideoJS播放器的模型，以指導您應捕獲哪些事件以及如何將數據提交到API。

其他功能

如果您需要其他功能，例如廣告或DRM解鎖，則還需要為播放器實現。查看源代碼Brightcove Player插件將提供一些有關您需要做什麼的有用線索。
