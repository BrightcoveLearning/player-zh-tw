簡介

進階 (頁內) 內嵌程式碼可讓您將播放程式直接存在於 HTML 頁面中，而不是在 iframe 中。這提供了輕鬆訪問播放器和相關的屬性和事件的好處。通過使用非 iframe 播放器，您將失去玩家的易用性作為社交媒體應用程序和共享中的一個單位，包含的單位。如需進一步資訊，請參閱選擇正確的內嵌程式碼文件。

用法

注意：如果您是 Brightcove 播放器客戶，您將不會使用該data-video-id屬性將視頻分配給您的播放器。您可以在播放器模塊中分配視頻或使用 curl。

若要使用頁面內嵌程式碼，請依照下列步驟執行：

  1. 使用  PLAYER 模組來建立玩家。
  2. 轉到 MEDIA 模塊並使用新創建的播放器發布視頻。
  3. 複製進階 內嵌程式碼。HTML 將類似於以下內容： 
      <video-js data-video-id="4093372393001"
    data-account="1507807800001"
    data-player="ry3j6rsze"
    data-embed="default"
    data-application-id
    class="video-js"
    controls></video-js>
  <script src="https://players.brightcove.net/1507807800001/ry3j6rsze_default/index.min.js"></script>
  4. 從瀏覽器複製 HTML，然後貼到完整body的 HTML 頁面。
  5. 瀏覽 HTML 頁面以查看播放器正常運作。

使用「媒體」模組發佈視訊時，您可以使用播放程式 URL 來預覽視訊，或是複製 iframe 或頁面內嵌程式碼，以貼到您的網頁或應用程式中。

注意：按一下預覽 URL 旁邊的「縮短」按鈕，即可產生縮短的預覽 URL。

標準連結會顯示 iframe 內嵌程式碼，而「進階」連結則會顯示頁內嵌程式碼。從播放程式管理 API 的角度來看，您也會看到稱為嵌入_in_page 實作的進階播放程式程式碼。

注意：有關使用響應式播放器的信息，請參閱Brightcove 播放器的響應式大小調整開發者文檔。

頁內嵌程式碼 (進階)

典型的頁面內嵌程式碼會顯示如下：

  <video-js data-video-id="5076962725001"
  data-account="1507807800001"
  data-player="BkTGbgSq"
  data-embed="default"
  data-application-id
  class="video-js"
  controls></video-js>
  <script src="https://players.brightcove.net/1507807800001/BkTGbgSq_default/index.min.js"></script>

雖然整合網頁內發佈程式碼可能比較複雜，但是當包含播放程式的頁面需要與播放程式通訊時，最好使用頁面內程式碼。何時使用頁面內嵌程式碼的一些範例包括：

  • 包含頁面中的代碼需要監聽播放器事件並採取行動
  • 播放器使用來自父頁面的樣式
  • iframe 代碼將導致應用程序邏輯失敗，就像從父頁面的重定向

即使您的最終實作未使用 iframe 嵌入程式 碼，您仍然可以將頁面內程式碼與 JavaScript 的外掛程式搭配使用，並為您的 CSS 使用個別檔案。這封裝了你的邏輯，以便您可以輕鬆地在多個玩家中使用它。

建議

除非某些應用程序邏輯需要使用頁面內代碼，否則使用 iframe 實現被認為是最佳實踐。 如果您使用 Audience 模組來追蹤檢視者參與度，則必須使用頁面內 (進階) 內嵌程式碼。

屬性

<video>標籤有許多屬性可供使用，以提供玩家應該如何行為的其他資訊。下表詳細說明可用屬性。

使用只能為 true 或 false (資料類型布林值 ) 的標籤屬性時，只要包含屬性 (沒有等號) 即可將其開啟。

右：controls

錯誤：controls="true"

為了使用布林值，在播放器配置設置中默認情況下應該禁用該屬性，然後通過將屬性包含在頁面內嵌代碼中啟用該屬性。例如，如果autoplay在播放程式設定中啟用，則無法在頁面內嵌程式碼中停用該功能。這表示如果您想要控制 Boolean 屬性，您希望確定它在播放程式設定中已停用，然後您可以選擇性地將適當的屬性放在頁面內嵌程式碼中加以啟用。在播放程式組態中啟用屬性會覆寫省略頁面內嵌程式碼中的屬性。

屬性 描述 資料類型
autoplay 表示播放器應該立即開始播放，在允許此操作的平台上。

如果設定autoplay為 Boolean 值，則會使用瀏覽器的原生自動播放行為。如果設定為下列三個有效字串值之一，就會發生下列情況：
  • play：玩家將手動嘗試調用播放。
  • muted：玩家將手動將播放器靜音並呼叫播放。
  • any：玩家會先嘗試呼叫播放。如果失敗，請將播放器靜音並呼叫播放。
如果上述任何一項失敗，玩家將顯示「大遊戲按鈕」，就好像自動播放設定為 false 一樣。

預先載入詳細資訊

注意：目前在 Studio Player 設置中設置的預載值將始終覆蓋標video籤屬性中設置的值，因此您必須在 Studio 中進行設置。
注意：預加載設置實際上只是 HTML5 規範中的建議，因此在不同瀏覽器上可能會有不同的行為。將值設定為「是」可能會對頻寬消耗產生重大影響。

預加載屬性通知瀏覽器視頻數據是否應該在加載視頻標籤後立即開始下載。選項包auto括metadata、和none。對於 HLS 播放技術，設置意味著稍微不同的事情，並在下面說明。

  • auto ( 預設值)：立即開始載入視訊 (如果瀏覽器同意)。部分行動裝置 (例如 iPhone 和 iPad) 不會預先載入視訊，以保護使用者' 頻寬。這就是為什麼這個值被稱為 auto，而不是像真實的更終的東西。

    對於 HLS 播放技術，自動表示播放程式下載 m3u8 資訊清單檔案，而只有 1 個區段而非 3-4，就像在 5.3 版本之前的 Brightcove 播放程式一樣。
  • metadata：只載入視訊的中繼資料，其中包括視訊的持續時間和尺寸等資訊。
    對於 HLS 播放技術，中繼資料與自動相同。
  • none：請勿預先載入任何視訊資料。這將等到用戶點擊播放開始下載。請注意，在 iOS 上，即使使用此設定，也可能會下載區段。如需詳細資訊，請參閱布萊特灣播放器和 iOS 文件。
    對於 HLS 播放技術，沒有意味著播放器只會下載 m3u8 清單，以便計算視頻持續時間。
  <video-js preload ...>
  or
  { "preload": "auto" }

注意事項：

  • 使用伺服器端廣告插入 (SSAI) 時，Brightcove 建議設定preload=none。如果設定preload為auto或metadata，則視訊的第一個影格串流，即使使用者沒有開始播放，也會將廣告計算為已傳送的廣告。
  • 由於 Internet Explorer 中的錯誤，使用 IE 時可能會忽略預先載入設定。
