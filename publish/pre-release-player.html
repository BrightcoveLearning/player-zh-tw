為什麼是預發布播放器？

您可能希望經常針對最新版本的 Brightcove Player 進行測試，包括預發布版本。本文檔向您展示瞭如何配置每次有新的預發布版本可用時都會更新的播放器。通過設置"version_alias": "prerelease"配置選項，您可以確定此播放器將始終具有最新的預發行版本。

在工作室的球員您可以將播放器設置為始終使用最新的預發布模塊：

這個怎麼運作

每當針對當前主要版本發布新的預發布版本時，播放器管理 API 都會重新發佈在其主分支上具有預發布版本別名的所有播放器。

使用API創建預發布播放器

以下 curl 語句使用 Player Management API 創建一個播放器，以始終使用最新的預發行版：

  curl \
    --header "Content-Type: application/json" \
    --user $EMAIL \
    --request POST \
    --data '{
        "name": "Player",
        "configuration": {
          "player": {
            "template": {
              "name": "single-video-template",
              "version_alias": "prerelease"
            }
          }
        }
      }' \
      https://players.api.brightcove.com/v1/accounts/$ACCOUNT_ID/players

注意：播放器配置GET對具有版本別名的播放器的請求將不包含關聯的播放器版本。但是，當收到具有預發行版本的播放器的預覽或發布請求時version_alias，則在發布之前，將從Player Management API數據庫中提取正確的預發行版本，並將其添加到播放器配置中。這意味著播放器版本可以在已發布的播放器中找到配置文件文件（如上圖所示）。以下是一個例子GET要求：
