概述：Brightcove播放器

{{ page.description }}

長處

播放器提供了一個平台，可在此平台上構建靈活，可擴展的視頻播放體驗。

主要優勢是：

  • 輕巧的HTML5視頻播放器
  • 豐富的插件環境可添加功能
  • 使用HTML，JavaScript和CSS自定義播放器外觀
  • 針對不同環境的兩種播放器實現
  • 豐富的功能集，例如提示點，字幕，本地化，廣告等。
  • 本機播放器SDK
  • 播放視頻雲庫視頻和具有Internet可訪問URL的視頻
This site in which you are viewing this document, https://player.support.brightcove.com, is for the web version of Brightcove Player.有關Brightcove Player的SDK版本的信息，請參見： https://sdks.support.brightcove.com/。另請注意，對 Web 版本的更改（例如樣式或特殊配置）不會自動對 SDK 版本進行更改，反之亦然。

發布影片

創建播放器時，您會看到三種用於發布播放器的實現。這些實現是：

  • 網址
  • 頁內嵌入
  • iframe

這些實現中的每一個都可以採用兩種方法。最常見的方法是使用動態播放器。該播放器可以播放許多不同的視頻。為此，視頻ID在運行時通過URL參數，視頻標記屬性或JavaScript API傳遞給播放器。從技術上講，這類似於可用於許多不同內容實例的模板網頁。這種方法的用例是通用視頻播放器，該視頻播放器根據用戶正在訪問的文章或視頻分配了不同的視頻。

一種更專業的方法是通過將視頻包含在播放器配置本身中來將視頻靜態綁定到播放器。此方法類似於創建靜態網頁。這樣做的好處是減少了播放器顯示海報圖像或開始播放視頻所必須進行的呼叫次數，從而提高了性能。這可以在Studio中完成，也可以直接使用API。這種方法的一個用例是使用播放器構建的產品頁面，其中包含詳細介紹單個產品的視頻。

筆記：儘管可以更改靜態綁定到播放器的視頻，但這不是一個好習慣。如果你這樣做， it is possible you will see the poster for the statically bound video briefly appear in the player before the video dynamically added is displayed .出於這個原因，創建一個最初沒有與之關聯的視頻的播放器是一種最佳做法，以便在視頻將動態更改的情況下使用。

球員管理

 

可以通過Studio或使用Player Management API以編程方式創建，自定義和管理播放器。

工作室

Studio提供了易於使用的界面來執行這些任務。

這Studio入門本文檔為學習Video Cloud Studio提供了一個很好的起點。

玩家管理 API

播放器管理API的功能分為兩大類。這些類別和詳細優勢在此處列出：

  • 集中管理
    • 自動版本更新
    • 通過API創建，更新和激活播放器
    • 一次更新數千名玩家
    • 播放器版本控制和回滾
  • 優化性能
    • 玩家資產的預編譯
    • 在播放器中嵌入縮略圖或視頻URL
    • 通過相同的CDN交付插件

有關播放器管理API的更多信息，請參見Brightcove播放器管理文檔。

發布靜態綁定的播放器

如上所述，您可以將視頻靜態綁定到播放器。這可以在播放器用戶界面中或通過播放器管理API來完成。有關信息，請參閱發布視頻和播放列表 與媒體模塊和一步步：球員管理文件。

在這些情況下，視頻ID不需要包含在iframe或直接鏈接中使用的視頻URL中，但必須包含在<video>頁內嵌入代碼。

玩家類型比較

Brightcove的播放器環境由三種不同的播放器類型組成。它們是：

  • Video.js：一個開放源代碼的JavaScript和CSS庫，它提供了一個框架，使它更易於使用並在HTML5視頻上構建。Video.js項目由Brightcove贊助，Brightcove為該項目做出了巨大貢獻。
  • Brightcove Player（Studio中的“播放器"模塊）：Brightcove開發的視頻播放器，使用Video.js作為基礎播放器框架。儘管您可以單獨使用Video.js，但大多數時候您都希望將其與其他插件和自定義項結合使用。Brightcove負責培養插件，管理其兼容性，並隨著時間的推移使它們保持最新。Brightcove播放器構成了所有Brightcove視頻體驗的基礎。
  • Brightcove播放器：用於創建和管理視頻播放器體驗的高性能服務。Brightcove旨在滿足全球領先的視頻發行商的需求，將Brightcove Player，領先的跨平台HTML5優先視頻播放器與強大的播放器管理API和性能優化服務集結合在一起。還包括Brightcove的Native iOS和Android SDK。

以下信息說明了為什麼選擇開源Video.js上的Brightcove Player可能對您有利。

  • 性能優化：播放器管理服務負責優化播放器的交付，通過預編譯插件，皮膚資源和縮略圖來最小化下載大小，從而優化每個播放器，然後通過高性能，全局優化的CDN交付它們。
  • 球員管理：播放器管理功能使用戶可以輕鬆跟踪播放器配置並進行更改。這對於具有多個播放器配置的客戶特別有用，包括不同的自定義和插件以及播放器樣式 - 適用於其站點或應用程序的不同部分。播放器管理服務允許他們更新這些播放器的配置，更改插件、樣式和其他設置 - 無需更改每個頁面的代碼。客戶只需簡單地重新發布播放器，這些更改就會自動推送到CDN。
  • 未來證明第三方集成：Brightcove與comScore，Nielsen，Google / Doubleclick，Freewheel等合作夥伴直接合作，以確保分析和廣告插件正常運行。此外，Brightcove通過瀏覽器和連接的設備保持最新趨勢，以確保我們的播放器在第三方Web技術的任何更新中都可以正常工作。Brightcove插件提供的某些功能在開源中不可用，例如使用Google的IMA SDK（用於顯示DFP廣告）在Flash和HTML5之間自動切換以顯示廣告。Brightcove確保插件已通過Brightcove Player進行了測試，但未通過Video.js的每個版本進行測試。
  • 行業領先的支持：選擇Brightcove Player的另一個重要原因是Brightcove支持該服務。總的來說，這意味著您作為客戶，擁有經驗豐富的合作夥伴，可以在您的每一個步驟中為您提供幫助。借助Brightcove，您可以更快地推向市場，並保證可以正常運行的系統，以及在不響應或有任何疑問時打電話給您的人。
  • 開發成本和上市時間：就像Brightcove所做的那樣，可以自己圍繞HTML視頻播放器構建所有功能，但是它並不便宜，並且會花費大量時間。此類工作可能不是Brightcove所出售公司的核心競爭力，因此，公司不可能像Brightcove所能做到的那樣或以更少的錢來做到這一點。開源一開始看起來比較便宜，但是要構建一個完整的解決方案需要更多的時間和精力，然後要保持播放器，插件和集成的最新維護成本。
  • 整合後：Brightcove Player專門用於支持對曾經編碼有服務器端廣告的流進行一次播放的流。此外，該播放器還與“曾經的用戶體驗"集成在一起，後者可基於VMAP內容提供豐富的交互式廣告體驗，從而告訴您何時在服務器端縫合廣告。
