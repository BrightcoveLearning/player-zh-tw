    學習指南：影片廣告

    本學習指南提供了一個起點，讓您瞭解影片廣告以及在 Brightcove 播放器中使用廣告。

    廣告基本概念

    在學習指南的這一部分，說明視頻廣告的基本概念。

    • 廣告類型
    • 廣告標準
    • 廣告連播
    • Google的IMA SDK
    • 影片廣告中的實體
    • Flash與HTML

    廣告類型

    有三種主要類型的視頻廣告，它們是：

    • 線性廣告：線性廣告會在播放影片期間發生，並在播放器本身中顯示。廣告會在使用者使用影片內容之前 (播放前)、(卷中) 或之後 (影片後) 使用中斷影片播放。以下範例影片示範如何使用 IMA 外掛程式。播放視頻以查看預滾動，在 5 秒鐘可跳過的中滾動，然後最後一個後卷。
    • 非線性廣告：播放時會刊登非線性廣告，讓使用者在檢視內容時看到廣告。非線性廣告可以是文字、圖形廣告，甚至是影片覆蓋。非線性廣告範例是在播放影片上顯示的文字重疊，但不會中斷影片。以下示例顯示屏幕截圖用戶可以單擊其中的一個覆蓋圖以轉到另一頁。覆蓋會出現一段指定的時間，但不會中斷視訊的播放。
    • 隨播廣告：隨附廣告是影片播放器周圍出現的文字或多媒體廣告，通常與影片內容相關聯。附屬廣告範例是針對影片內容調整的橫幅橫幅。以下示例顯示屏幕截圖隨播廣告的廣告位的位置，用紅色的虛線標出並填充了橫幅。

    廣告標準

    互動廣告局（IAB）是一家由超過 650 家主要媒體和科技公司組成的廣告業務組織，致力於制定行業標準，進行研究，並為線上廣告行業提供法律支持。

    的IAB視頻套件視頻在解釋原因，方式或標準方面做得非常出色。

    以下是影片廣告的主要標準：

    • 視頻多個廣告播放列表（VMAP）：說明影片的預錄、中間、後捲和非線性廣告，以及應該顯示影片的時間。
      • 從 IAB 的標準來看：「IAB 影片多重廣告播放清單 (VMAP) 規格是一種 XML 範本，視訊內容擁有者可以用來說明廣告庫存插入的結構，當視訊播放器或內容發佈出口無法控制廣告空間時。「
    • 視頻廣告投放模板（VAST）：VMAP 說有一個預滾動，VAST 描述了預滾。
      • 從 IAB 的標準來看：「影片廣告放送範本 (VAST) 包含串流內影片的標準 XML 廣告回應，以及適用於開發人員的 XML 結構描述定義 (「XSD」)。它旨在適應線上數位視訊廣告業務目前大部分的做法。「
    • 視頻播放器廣告服務界面定義（VPAID）：對於互動式廣告來說，V付費就像是 VAST。
      • 已付款：根據 IAB 的標準」「影片播放器廣告服務介面定義 (VPAYE) 建立了影片播放器和廣告單元之間的共同介面，提供豐富的串流內廣告體驗。「
      • 一個簡單的VPAID示例將在其中顯示簡短的介紹性視頻，並將鼠標懸停在更多信息播放更長的視頻時會發短信。
      • VPAED 格式通常被代理商和廣告商用於運行可視性腳本。

    看到IAB的網站完整的標准文檔。

    廣告網格

    VAST 3.x 中的新功能是廣告網繭的概念。廣告網繭是一組連續線性廣告的傳送。下圖代表一個廣告網繭，其中包含三個廣告，這些廣告會在一般影片內容之前依序播放。廣告 Pods 可以在內容影片播放之前、休息期間或播放後播放，並可像電視廣告片段一樣運作，包含多個廣告點。

    您可以使用 VMAP 指定廣告網格的分隔點。

    在裡面廣告事件和廣告對象您會看到一些有關播放廣告連播的事件。

    谷歌的 IMA SDK

    Google 是影片廣告的主要參與者，並定義了 Google 互動式媒體廣告 (IMA) SDK，可讓發佈商在影片和遊戲中顯示線性、非線性和伴侶廣告。SDK 允許開發者向 DoubleClick 發佈商 (DFP)、谷歌 AdSense 聯播網或任何符合 VEST 規範的廣告伺服器提出視訊廣告請求。為布萊特灣播放器的 IM3 插件整合了布萊特灣播放器與谷歌的互動媒體廣告（IMA）版本 3 為 HTML5 和快閃記憶體。這可讓您為您的布萊特灣播放器請求和追蹤廣告。

    要瞭解的一些基本 IMA 術語如下：

    • 廣告規則：定義廣告在影片內容中顯示的時間、位置和方式。您可以建立廣告規則，決定廣告在影片中或工作階段中播放的時間 (跨多部影片串流)、多久以及觸發的結果。廣告規則與 VMAP 文件大致相似。
    • 廣告標籤：給出指示，瀏覽器從廣告服務器檢索廣告，以及如何處理廣告。該指示將產生一個廣告請求，並會有從廣告服務器適當的響應。廣告標籤與 VAST 文件大致相似。

    有關IMA的更多信息，請參見互動媒體廣告在Google開發者幫助部分。

    影片廣告中的實體

    雖然有許多參與視頻廣告的實體分組，但是一個非常簡單的例子如下：

    • 發佈商-您可以顯示影片內容，並希望使用廣告來獲得收益。
    • 廣告客戶-有他們希望客戶/潛在客戶觀看的廣告。
    • 廣告聯播網-比對出版商的廣告空間與廣告商的廣告。
    • 視訊技術供應商-為發佈商和廣告網路提供編碼和播放器技術。布萊特灣適合視訊技術供應商小組。

    快閃記憶體與 HTML

    目前 Flash 已經成為最常用的廣告格式，讓位給 HTML5。因此，建議針對Brightcove Player的IMA3插件adTechOrder選項設置為html5首先，然後閃。有關更多信息，請參見使用IMA的Flash不再有效。

    與布萊特灣播放器的廣告工作流程

    讓廣告與 Brightcove 播放器合作時，基本的廣告工作流程是：

    1. 聯繫廣告提供商。所有與廣告相關的Brightcove合作夥伴都可以在此找到搜索從合作夥伴頁面。這些廣告供應商與布萊特灣創建插件來支持他們的廣告技術是谷歌的 IMA3 技術和 Freewheel。許多其他廣告供應商已經為布萊特灣播放器創建了自己的插件。下表提供了不同廣告供應商和廣告相關技術的一些細節，以及它們如何與 Brightcove 播放器的關係。提供者/技術 IMA3插件。
      提供者/技術 評論
      連按兩下 (IM3) 通過支持IMA3插件。
      飛輪 通過支持FreeWheel插件
      斑點 您可以直接從 SpotX 使用外掛程式。查看SpotX的Brightcove插件集成文檔頁面。
      第三方 VMAP（通過 IM3 插件） VMAP 支援前捲、中捲、捲後卷和同伴。
      第三方 VAST3（通過 IM3 插件） API 可透過 VAST 網址插入廣告。
      VPAD 2 廣告 (通過 IMA3 插件) 布萊特灣播放器支持 Javascript VPAD 2.0 廣告單元，這意味著它可以在所有設備上不使用 Flash 在任何目的的的情況下進行部署。

      使用服務器端廣告插入（SSAI）。您也可以使用 Brightcove 的 SSAI 外掛程式，將廣告拼接到您的影片串流中。此功能可透過「動態傳遞」啟用。如需詳細資訊，請參閱下列文件：

      • Video Cloud SSAI概述
      • 使用 SSAI 外掛程式進行廣告

       

    2. 驗證您的廣告代碼。您的廣告供應商將為您提供一個 URL，您將提供布萊特灣播放器。建議您在開始整合播放器之前，先確定廣告代碼正常運作。如果您的廣告提供商使用標準的VAST或VMAP廣告響應，則可以使用Google的視頻套件檢查器快速測試您的廣告代碼是否正常運行。其他一些廠商提供類似的技術。驗證網址不僅是初始設定時的最佳做法，而且如果稍後發生問題，排解廣告播放的第一步就是重新測試網址。
    3. 使用供應商的信息配置Brightcove Player。驗證廣告供應商網址的有效性後，請使用適當的外掛程式將 Brightcove 播放器連接到廣告伺服器。
