廣告類型

視頻廣告主要有三種類型，分別是：

  • 線性廣告：線性廣告會在視頻播放期間發生，並顯示在播放器本身中。廣告會在用戶消費視頻內容之前（前片），中間（中間片）或之後（後片）中斷視頻播放。以下示例視頻演示了IMA插件的用法。播放視頻以觀看前貼片廣告，在5秒鐘時可跳過的中插片，最後觀看後貼片。
  • 非線性廣告：非線性廣告會在播放過程中投放，因此用戶在查看內容時會看到廣告。非線性廣告可以是文字廣告，圖形廣告，甚至可以是視頻重疊廣告。非線性廣告的一個示例是在廣告視頻上顯示但不會中斷視頻的文字疊加層。以下示例顯示屏幕截圖用戶可以單擊一個覆蓋圖以轉到另一個頁面。疊加層會顯示指定的時間，但不會中斷視頻的播放。
  • 隨播廣告：隨播廣告是出現在視頻播放器周圍的文字或展示廣告，通常與視頻內容相關。隨播廣告的一個示例是橫幅，該橫幅已調整為視頻內容。以下示例顯示屏幕截圖隨播廣告的廣告位的位置，用紅色的虛線標出，並充滿了橫幅。

廣告標準

交互式廣告局（IAB）是由650多家領先的媒體和技術公司組成的廣告業務組織，致力於開發行業標準，進行研究並為在線廣告行業提供法律支持。

這IAB視頻套件視頻在解釋原因，方式或標準方面做得非常出色。

以下是視頻廣告的關鍵標準：

  • 視頻多個廣告播放列表（VMAP）：描述視頻的前置廣告，中插廣告，後置廣告和非線性廣告，以及應在何時顯示。
    • 根據IAB的標準："The IAB Video Multiple Ad Playlist (VMAP) specification is an XML template that video content owners can use to describe the structure for ad inventory insertion when they don’t control the video player or the content distribution outlet."
  • 視頻廣告投放模板（VAST）：VMAP表示有一個預卷，VAST描述了該預卷。
    • 根據IAB的標準："The Video Ad Serving Template (VAST) includes a standard XML-based ad response for in-stream video as well as an XML Schema Definition (“XSD”) for developers.它旨在適應在線數字視頻廣告業務中的大多數當前做法。"
  • 視頻播放器廣告服務界面定義（VPAID）：VPAID就像交互式廣告的VAST一樣。
    • 已付款：根據IAB的標準"，“視頻播放器廣告服務接口定義（VPAID）在視頻播放器和廣告單元之間建立了通用接口，從而提供了豐富的交互式流內廣告體驗。"
    • 一個簡單的VPAID示例就是顯示簡短的介紹性視頻，並且如果您將鼠標懸停在更多信息播放更長的視頻時會發短信。
    • 代理商和廣告客戶通常使用VPAID格式來運行可見性腳本。

看IAB的網站完整的標准文檔。

廣告連播

VAST 3.x的新功能是廣告連播的概念。廣告連播是一系列順序線性廣告的投放。下圖顯示了一個廣告連播，其中包含三個在正常視頻內容之前順序播放的廣告。廣告盒可以在內容視頻播放之前，插入期間或之後播放，其功能類似於帶有多個廣告點的電視廣告播放。

您可以使用VMAP指定廣告連播的休息時間。

在裡面廣告事件和廣告對象您會看到一些有關播放廣告連播的事件。

Google的IMA SDK

Google是視頻廣告的主要參與者，並定義了Google交互式媒體廣告（IMA）SDK，使發布商可以在視頻和遊戲中顯示線性，非線性和隨播廣告。通過SDK，開發人員可以向DFP廣告管理系統（DFP），Google AdSense網絡或任何符合VAST的廣告服務器發出視頻廣告請求。用於Brightcove Player的IMA3插件將Brightcove Player與適用於HTML5和Flash的Google交互式媒體廣告（IMA）版本3集成在一起。這使您可以請求和跟踪Brightcove Player的廣告。

一些基本的IMA術語需要理解：

  • 廣告規則：定義廣告與您的視頻內容一起顯示的時間，位置和方式。您可以創建廣告規則，以確定廣告何時在視頻中或會話中播放（跨多個視頻流），播放多長時間以及由哪個觸發。廣告規則與VMAP文檔大致相似。
  • 廣告代碼：向瀏覽器提供指令以從廣告服務器檢索廣告以及如何處理廣告。這些說明將生成廣告請求，並且廣告服務器會做出適當的響應。廣告代碼與VAST文檔大致相似。

有關IMA的更多信息，請參見互動媒體廣告在Google開發者幫助部分。

影片廣告中的實體

儘管存在與視頻廣告有關的實體的許多分組，但以下是一個非常簡單的示例：

  • 發布商-可以顯示視頻內容，並希望使用廣告來獲得收益。
  • 廣告商-擁有他們希望客戶/潛在客戶觀看的廣告。
  • 廣告網絡 - 匹配發布商給廣告商的廣告空間廣告。
  • 視頻技術提供商-為發布商和廣告網絡提供編碼和播放器技術。Brightcove適用於視頻技術提供商組。

Flash與HTML

目前，Flash正在讓HTML5成為最常用的廣告格式。因此，建議針對Brightcove Player的IMA3插件adTechOrder選項設置為html5首先，然後閃光。有關更多信息，請參見使用IMA的Flash不再有效。

Brightcove Player的廣告工作流程

使廣告與Brightcove Player配合使用時的基本廣告工作流程是：

  1. 聯繫廣告提供商。所有與廣告相關的Brightcove合作夥伴都可以在此找到搜索從合作夥伴頁面。那些使用Brightcove創建的插件來支持其廣告技術的廣告提供商是Google的IMA3技術和FreeWheel。其他許多廣告提供商也為Brightcove Player創建了自己的插件。下表詳細介紹了不同的廣告提供商和廣告相關技術，以及它們與Brightcove Player的關係。提供者/技術 IMA3插件。
    提供者/技術 評論
    DoubleClick（IMA3） 通過支持IMA3插件。
    飛輪 通過支持FreeWheel插件
    SpotX 可直接從SpotX獲得插件。查看SpotX的Brightcove插件集成文檔頁面。
    第三方VMAP（通過IMA3插件） VMAP支持前插，中插，後插和隨播廣告。
    第三方VAST3（通過IMA3插件） API可用於通過VAST URL注入廣告。
    VPAID 2廣告（通過IMA3插件） Brightcove Player支持Javascript VPAID 2.0廣告單元，這意味著可以在不使用Flash的情況下在所有設備上進行部署。

    使用服務器端廣告插入（SSAI）。您還可以使用Brightcove的SSAI插件將廣告拼接到視頻流中。動態交付啟用了此功能。有關詳細信息，請參閱以下文檔：

    • Video Cloud SSAI概述
    • 使用SSAI插件做廣告
  2. 驗證您的廣告代碼。您的廣告提供商將為您提供一個URL，您將提供Brightcove Player。在開始播放器集成之前，請確保您的廣告代碼能夠正常工作。如果您的廣告提供商使用標準的VAST或VMAP廣告響應，則可以使用Google的視頻套件檢查器快速測試您的廣告代碼是否正常運行。其他一些供應商也提供類似的技術。驗證URL不僅是初始設置時的最佳做法，而且如果以後出現問題，則對廣告播放進行故障排除的一個很好的第一步就是重新測試URL。
  3. 使用供應商的信息配置Brightcove Player。驗證廣告提供商URL的有效性之後，使用適當的插件將Brightcove Player連接到廣告服務器。
  4. 測試配置。您將要確保廣告可以在所有預期的平台，操作系統和瀏覽器上播放。例如，如果您的廣告提供商僅提供Flash廣告，則iOS設備上可能會出現問題。另外，默認的桌面瀏覽器配置可以更改廣告行為。
